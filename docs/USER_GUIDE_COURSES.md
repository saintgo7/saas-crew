## 코스 시스템 소개

### 코스란?

**코스(Course)**는 특정 기술이나 주제를 체계적으로 학습할 수 있는 교육 과정입니다.

**코스의 구성**:
- 여러 개의 챕터(Chapter)로 구성
- 각 챕터는 독립적인 학습 단위
- 이론 설명 + 실습 + 과제
- 진도 추적 및 완강 인증

### 코스 레벨 시스템

코스는 3가지 난이도로 분류됩니다.

#### JUNIOR (초급)

**대상**: 프로그래밍을 처음 시작하는 학생

**특징**:
- 기본 개념 위주
- 천천히 단계적 학습
- 많은 예제와 설명
- 쉬운 과제

**예시 코스**:
- HTML/CSS 기초
- JavaScript 입문
- Git 시작하기
- 프로그래밍 기초

#### SENIOR (중급)

**대상**: 기본 문법을 익힌 학생

**특징**:
- 실전 기술 위주
- 프레임워크/라이브러리
- 프로젝트 기반 학습
- 중간 난이도 과제

**예시 코스**:
- React로 웹앱 만들기
- Node.js 백엔드 개발
- PostgreSQL 데이터베이스
- RESTful API 설계

#### MASTER (고급)

**대상**: 실무 경험이 있는 학생

**특징**:
- 고급 아키텍처
- 성능 최적화
- 확장 가능한 시스템 설계
- 복잡한 프로젝트

**예시 코스**:
- 마이크로서비스 아키텍처
- Kubernetes 실전
- 시스템 설계 패턴
- 대규모 트래픽 처리

### 코스 학습 혜택

#### XP 획득

- 챕터 완료: 10-50 XP
- 코스 완강: 100-500 XP
- 과제 제출: 추가 XP

#### 스킬 증명

- 완강 뱃지 획득
- 프로필에 표시
- 포트폴리오 강화

#### 체계적 성장

- 학습 경로 제시
- 진도 관리
- 목표 달성 동기 부여

## 코스 둘러보기

### Step 1: 코스 페이지 접속

상단 네비게이션 메뉴에서 **"코스"** 클릭

```
[스크린샷 자리: 코스 목록 페이지]
```

### Step 2: 코스 필터링

원하는 코스를 빠르게 찾기 위해 필터를 사용합니다.

#### 레벨별 필터

```
[전체] [Junior] [Senior] [Master]
```

현재 나의 랭크에 맞는 코스를 선택하세요.

**추천 학습 경로**:
1. Junior 랭크: Junior 코스 위주
2. Senior 랭크: Senior 코스 위주 + Junior 복습
3. Master 랭크: Master 코스 + 원하는 Senior 코스

#### 카테고리별 필터

```
[Frontend] [Backend] [Database] [DevOps] [Mobile] [AI/ML]
```

관심 분야에 따라 선택하세요.

#### 태그별 필터

```
#React #Node.js #Python #Docker #TypeScript
```

특정 기술 스택을 학습하고 싶을 때 사용합니다.

코스 목록의 정렬 순서를 변경할 수 있습니다.

**정렬 옵션**:
- **최신순**: 최근에 개설된 코스
- **인기순**: 수강생이 많은 코스
- **난이도순**: Junior → Senior → Master
- **완강률순**: 완강률이 높은 코스 (추천)

```

```

### 진도율 계산 방식

진도율은 완료한 챕터 수를 기준으로 계산됩니다.

```
진도율 = (완료한 챕터 수 / 전체 챕터 수) × 100
```

**예시**:
- 전체 챕터: 10개
- 완료한 챕터: 6개
- 진도율: 60%

#### 텍스트 콘텐츠
```markdown
# useState Hook 이해하기

useState는 함수형 컴포넌트에서 상태를 관리하는
가장 기본적인 Hook입니다.

## 기본 문법

​```javascript
const [state, setState] = useState(initialValue);
​```

- state: 현재 상태 값
- setState: 상태를 업데이트하는 함수
- initialValue: 초기값
```

#### 코드 예제
```javascript
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        증가
      </button>
    </div>
  );
}
```

**기능**:
- 코드 복사 버튼
- 실행 가능한 샌드박스 (CodeSandbox, StackBlitz)
- 문법 하이라이팅

## 학습 팁

### 효과적인 학습 방법

#### 1. 꾸준히 학습하기

**하루 30분 규칙**:
- 매일 최소 30분 이상 학습
- 짧더라도 꾸준히 하는 것이 중요
- 습관 형성에 집중

#### 2. 실습 위주로 학습

**이론 30%, 실습 70%**:
- 이론을 읽은 후 바로 실습
- 예제 코드를 직접 작성
- 변형해보며 실험

#### 3. 메모하기

**학습 노트 작성**:
- 중요한 개념 정리
- 에러 해결 방법 기록
- 나만의 치트시트 만들기

#### 4. 복습하기

**에빙하우스 망각 곡선 활용**:
- 학습 후 1일 뒤 복습
- 1주 뒤 복습
- 1개월 뒤 복습

### 학습 속도 관리

#### 자신의 페이스 유지

**무리하지 않기**:
- 하루에 2-3챕터가 적당
- 완벽히 이해한 후 다음 챕터로
- 막히면 커뮤니티에 질문

#### 기한 설정

**자율 학습이지만 목표는 필요**:
- "이번 주에 5챕터 완료"
- "이번 달에 코스 완강"
- 달력에 표시하고 체크

### 커뮤니티 활용

#### 질문하기

**막히면 바로 질문**:
- 커뮤니티 Q&A 활용
- 구체적으로 질문
- 시도한 방법 공유

#### 스터디 그룹

**함께 학습하기**:
- 같은 코스 듣는 크루원 찾기
- 주간 진도 공유
- 서로 격려하고 동기 부여

## 자주 묻는 질문

### Q1. 코스를 순서대로 들어야 하나요?

**A**: 아니요. 원하는 코스를 자유롭게 선택할 수 있습니다. 다만, 사전 요구사항이 있는 코스는 먼저 기초 코스를 듣는 것을 추천합니다.

### Q2. 여러 코스를 동시에 들을 수 있나요?

**A**: 가능합니다. 하지만 집중력을 위해 2-3개 이내로 제한하는 것을 추천합니다.

### Q3. 챕터를 건너뛰어도 되나요?

**A**: 일반적으로 가능합니다. 하지만 일부 코스는 순차 학습이 필요하며, 이 경우 이전 챕터를 완료해야 다음 챕터가 잠금 해제됩니다.

### Q4. 과제를 꼭 제출해야 하나요?

**A**: 선택 사항입니다. 과제 제출 없이도 챕터를 완료할 수 있습니다. 하지만 과제를 통해 더 깊이 학습하고 추가 XP를 얻을 수 있습니다.

### Q5. 완강한 코스를 다시 들을 수 있나요?

**A**: 가능합니다. 복습용으로 언제든지 다시 학습할 수 있습니다. 다만 추가 XP는 지급되지 않습니다.

### Q6. 코스 완강 후 수료증을 받을 수 있나요?

**A**: 현재는 프로필 뱃지만 제공됩니다. 향후 PDF 수료증 발급 기능이 추가될 예정입니다.
