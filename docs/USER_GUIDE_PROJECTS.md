## 프로젝트란?

### 프로젝트의 의미

WKU Software Crew에서 **프로젝트**는 여러분이 직접 기획하고 개발한 소프트웨어 작품입니다.

**프로젝트의 목적**:
- 학습한 내용을 실습으로 적용
- 포트폴리오 구축
- 다른 크루원과 협업 경험
- XP 획득 및 레벨업
- 기업 채용 시 증빙 자료

### 프로젝트 유형

#### 개인 프로젝트
- 혼자 진행하는 프로젝트
- 자신의 아이디어를 자유롭게 구현
- 빠른 의사결정 가능

#### 팀 프로젝트
- 2명 이상이 함께 진행
- 역할 분담 (Frontend, Backend, Design)
- 협업 경험 축적
- 더 큰 규모의 프로젝트 가능

### 프로젝트 예시

**Junior 레벨 프로젝트**:
- 할 일 관리 앱 (To-Do List)
- 계산기 앱
- 날씨 정보 앱
- 간단한 블로그

**Senior 레벨 프로젝트**:
- 풀스택 SNS 앱
- 전자상거래 플랫폼
- 실시간 채팅 앱
- 프로젝트 관리 도구

**Master 레벨 프로젝트**:
- 확장 가능한 마이크로서비스 아키텍처
- AI/ML 기반 추천 시스템
- 대규모 데이터 처리 파이프라인
- 오픈소스 라이브러리 기여

## 프로젝트 등록 방법

### Step 1: 프로젝트 페이지 이동

1. 상단 네비게이션 메뉴에서 **"프로젝트"** 클릭
2. 우측 상단의 **"새 프로젝트"** 버튼 클릭

```
[스크린샷 자리: 프로젝트 목록 페이지 - 새 프로젝트 버튼]
```

### Step 2: 기본 정보 입력

#### 프로젝트 이름 (필수)

프로젝트의 명확한 이름을 입력합니다.

**좋은 예시**:
- "할 일 관리 앱 - TaskMaster"
- "실시간 채팅 플랫폼 - ChatHub"
- "날씨 정보 대시보드 - WeatherNow"

**나쁜 예시**:
- "프로젝트1" (의미 없음)
- "테스트" (임시 성격)
- "aaa" (불명확)

#### 슬러그 (URL 주소)

프로젝트 URL에 사용될 고유 주소입니다.

- 영문 소문자, 숫자, 하이픈(-)만 사용
- 공백은 하이픈으로 자동 변환
- 예시: `taskmaster`, `chat-hub`, `weather-now`

**URL 예시**:
```
https://wku-crew.com/projects/taskmaster
https://wku-crew.com/projects/chat-hub
```

#### 프로젝트 설명 (선택, 권장)

프로젝트의 목적, 기능, 특징을 설명합니다.

**작성 가이드**:
```markdown
# 프로젝트 개요
할 일을 효율적으로 관리하는 웹 애플리케이션입니다.

## 주요 기능
- 할 일 추가, 수정, 삭제
- 카테고리별 분류
- 마감일 알림
- 진행 상황 시각화

## 사용 기술
- Frontend: React, TailwindCSS
- Backend: Node.js, Express
- Database: PostgreSQL
```

**팁**: Markdown 문법을 사용하면 보기 좋게 표시됩니다.

### Step 3: 기술 스택 선택

프로젝트에서 사용한 기술을 태그로 추가합니다.

**추가 방법**:
1. 입력 필드에 기술 이름 입력
2. Enter 키 또는 "추가" 버튼 클릭
3. 여러 개 추가 가능

**예시**:
```
[React] [TypeScript] [Node.js] [PostgreSQL] [Docker]
```

**자주 사용되는 태그**:
- Frontend: React, Vue, Angular, Next.js, HTML/CSS
- Backend: Node.js, Python, Java, Go
- Database: PostgreSQL, MongoDB, MySQL
- DevOps: Docker, Kubernetes, AWS, GCP
- Mobile: React Native, Flutter

### Step 4: GitHub 저장소 연동 (선택, 권장)

GitHub 저장소 URL을 입력하면 자동으로 프로젝트 정보를 가져옵니다.

**입력 형식**:
```
https://github.com/username/repository-name
```

**자동으로 가져오는 정보**:
- README.md 내용
- 사용 언어 (Language stats)
- 최근 커밋 정보
- Stars, Forks 수

**주의사항**:
- 공개 저장소(Public Repository)만 연동 가능
- 비공개 저장소는 향후 지원 예정

### Step 5: 배포 URL 입력 (선택)

프로젝트를 배포했다면 실제 동작하는 URL을 입력합니다.

**예시**:
```
https://taskmaster.vercel.app
https://my-project.netlify.app
https://my-app.herokuapp.com
```

**배포 플랫폼 추천**:
- Vercel: Next.js, React 프로젝트
- Netlify: Static 사이트, SPA
- Heroku: 백엔드 API
- Cloudflare Pages: 정적 사이트
- AWS, GCP: 복잡한 프로덕션 환경

### Step 6: 커버 이미지 업로드 (선택)

프로젝트를 대표하는 이미지를 업로드합니다.

**이미지 종류**:
- 프로젝트 스크린샷
- 로고 이미지
- UI/UX 디자인
- 아키텍처 다이어그램

**이미지 요구사항**:
- 파일 형식: JPG, PNG, GIF
- 최대 크기: 10MB
- 권장 비율: 16:9 (예: 1920x1080)

**팁**: 여러 화면을 한 이미지로 합쳐서 올리면 효과적입니다.

```
[스크린샷 자리: 프로젝트 등록 폼 - 전체 화면]
```

### Step 7: 가시성 설정

프로젝트의 공개 범위를 선택합니다.

- **PUBLIC (공개)**: 모든 사람이 볼 수 있음
- **PRIVATE (비공개)**: 나만 볼 수 있음
- **TEAM (팀 전용)**: 팀원만 볼 수 있음

**권장 설정**:
- 완성된 프로젝트: PUBLIC
- 진행 중인 프로젝트: TEAM 또는 PRIVATE
- 포트폴리오용: PUBLIC

### Step 8: 저장 및 확인

1. **"프로젝트 생성"** 버튼 클릭
2. 성공 메시지 확인: "프로젝트가 생성되었습니다!"
3. 자동으로 프로젝트 상세 페이지로 이동

**축하합니다!** 첫 프로젝트가 등록되었습니다.

**획득 XP**: 100 XP (첫 프로젝트 등록)

## 프로젝트 관리

프로젝트를 등록한 후에도 언제든지 수정하거나 삭제할 수 있습니다.

### 프로젝트 수정

#### Step 1: 프로젝트 상세 페이지 이동

1. "내 프로젝트" 목록에서 수정할 프로젝트 클릭
2. 우측 상단의 **"편집"** 버튼 클릭

#### Step 2: 정보 수정

- 프로젝트 이름
- 설명
- 기술 스택
- GitHub 저장소 URL
- 배포 URL
- 커버 이미지
- 가시성 설정

**주의**: 슬러그(URL)는 수정할 수 없습니다. 잘못 입력한 경우 프로젝트를 삭제하고 다시 생성해야 합니다.

#### Step 3: 저장

**"변경사항 저장"** 버튼을 클릭합니다.

**획득 XP**: 20 XP (주 1회 제한)

### 프로젝트 업데이트 공유

프로젝트에 큰 변화가 있을 때 크루원들에게 알릴 수 있습니다.

**업데이트 예시**:
- 새로운 기능 추가
- 버전 업데이트 (v1.0 → v2.0)
- 배포 완료
- 주요 버그 수정

**공유 방법**:
1. 프로젝트 편집 페이지에서 설명 수정
2. 커뮤니티에 업데이트 소식 게시
3. 프로필 피드에 자동으로 표시

### 프로젝트 삭제

#### 주의사항

**프로젝트 삭제는 되돌릴 수 없습니다!**

- 프로젝트 정보 완전 삭제
- 팀원 관계 모두 해제
- 관련 통계 데이터 삭제
- GitHub 저장소는 영향 없음 (연동만 해제)

#### 삭제 절차

1. 프로젝트 상세 페이지 하단의 **"위험 영역"** 섹션으로 스크롤
2. **"프로젝트 삭제"** 버튼 클릭
3. 확인 대화상자에서 프로젝트 이름 입력
4. **"삭제 확인"** 버튼 클릭

```
[스크린샷 자리: 프로젝트 삭제 확인 대화상자]
```

## 팀원 추가하기

팀 프로젝트를 진행하려면 다른 크루원을 팀원으로 초대할 수 있습니다.

### 팀원 역할 시스템

WKU Software Crew는 4가지 역할을 지원합니다.

#### OWNER (소유자)

- 프로젝트를 생성한 사람
- 모든 권한 보유
- 프로젝트 삭제 가능
- 역할 변경 불가 (항상 Owner)

**권한**:
- 프로젝트 수정
- 프로젝트 삭제
- 팀원 추가/제거
- 모든 역할 부여 가능

#### ADMIN (관리자)

- 프로젝트 관리 권한
- 팀원 관리 가능
- 프로젝트 삭제 불가

**권한**:
- 프로젝트 수정
- 팀원 추가/제거
- MEMBER, VIEWER 역할 부여 가능
- ADMIN 역할은 Owner만 부여 가능

#### MEMBER (멤버)

- 일반 팀원
- 프로젝트에 기여하는 개발자

**권한**:
- 프로젝트 내용 보기
- 코드 기여 (GitHub 기준)
- 프로젝트 수정 불가

#### VIEWER (뷰어)

- 읽기 전용 권한
- 프로젝트를 관찰하는 역할

**권한**:
- 프로젝트 내용 보기만 가능
- 수정, 기여 불가

### 팀원 추가 절차

#### Step 1: 팀 관리 페이지 이동

1. 프로젝트 상세 페이지에서 **"팀"** 탭 클릭
2. **"팀원 추가"** 버튼 클릭

```
[스크린샷 자리: 팀 관리 페이지]
```

#### Step 2: 크루원 검색

1. 검색창에 이름 또는 이메일 입력
2. 자동 완성 목록에서 추가할 크루원 선택

**검색 팁**:
- 부분 검색 가능 (예: "홍" → "홍길동" 검색)
- 이메일로도 검색 가능
- GitHub 사용자 이름으로도 검색 가능

#### Step 3: 역할 선택

추가할 크루원의 역할을 선택합니다.

**역할 선택 가이드**:
- 함께 개발하는 동료: **MEMBER**
- 프로젝트를 관리할 리더: **ADMIN**
- 멘토나 조언자: **VIEWER**

#### Step 4: 초대 보내기

**"초대 보내기"** 버튼을 클릭합니다.

- 상대방에게 알림 발송
- 이메일로도 통지 (설정에 따라)

#### Step 5: 초대 수락 대기

상대방이 초대를 수락하면 팀원으로 추가됩니다.

**초대 유효 기간**: 7일

### 팀원 역할 변경

#### 권한 있는 사용자
- Owner: 모든 역할 변경 가능
- Admin: MEMBER, VIEWER 간 변경 가능

#### 변경 절차
1. 팀 관리 페이지에서 변경할 팀원 찾기
2. 역할 드롭다운 메뉴 클릭
3. 새로운 역할 선택
4. 자동 저장

```
[스크린샷 자리: 팀원 역할 변경 드롭다운]
```

### 팀원 제거

#### 주의사항
- 제거된 팀원은 프로젝트 접근 불가 (가시성이 TEAM인 경우)
- 다시 초대 가능

#### 제거 절차
1. 팀 관리 페이지에서 제거할 팀원 찾기
2. **"제거"** 버튼 클릭
3. 확인 대화상자에서 **"확인"** 클릭

## 프로젝트 쇼케이스

완성된 프로젝트는 쇼케이스에 등록하여 많은 크루원에게 보여줄 수 있습니다.

### 쇼케이스란?

**쇼케이스**는 완성도 높은 프로젝트를 모아놓은 갤러리입니다.

**특징**:
- 메인 페이지에 featured 표시
- 높은 조회수 및 좋아요 획득
- 기업 담당자에게 노출
- 포트폴리오로 활용

### 쇼케이스 등록 조건

자동으로 쇼케이스에 등록되지 않습니다. 다음 조건을 충족해야 합니다.

**필수 조건**:
- 가시성이 PUBLIC일 것
- GitHub 저장소 연동 또는 배포 URL 존재
- 프로젝트 설명이 100자 이상
- 커버 이미지 등록

**권장 조건**:
- README.md에 상세 설명
- 실제 동작하는 배포 링크
- 기술 스택 태그 3개 이상
- 프로젝트 완성도 80% 이상

### 쇼케이스 등록 방법

#### Step 1: 프로젝트 완성도 확인

프로젝트 상세 페이지에서 완성도 체크리스트를 확인합니다.

```
✅ 공개 설정 완료
✅ GitHub 저장소 연동
✅ 배포 URL 등록
✅ 프로젝트 설명 작성
✅ 커버 이미지 업로드
✅ 기술 스택 태그 추가
```

#### Step 2: 쇼케이스 신청

1. 프로젝트 편집 페이지로 이동
2. **"쇼케이스 신청"** 버튼 클릭
3. 신청 사유 작성 (선택)

```
[스크린샷 자리: 쇼케이스 신청 양식]
```

#### Step 3: 승인 대기

관리자가 프로젝트를 검토하고 승인합니다.

**검토 기준**:
- 프로젝트 완성도
- 코드 품질
- 실제 동작 여부
- 독창성 및 유용성

**검토 기간**: 보통 3-5일

#### Step 4: 승인 시 혜택

**쇼케이스에 등록되면**:
- 메인 페이지 featured 섹션에 표시
- 조회수 대폭 증가
- 크루원들의 좋아요 및 피드백
- XP 보너스 획득 (200 XP)

### Featured 프로젝트 선정

쇼케이스 중에서도 특히 우수한 프로젝트는 **Featured** 뱃지를 받습니다.

**선정 기준**:
- Senior 레벨 이상
- 좋아요 50개 이상
- 조회수 500회 이상
- 코드 품질 우수
- 실용성 및 완성도

**Featured 혜택**:
- 메인 페이지 최상단 노출
- 기업 담당자 직접 연락 가능
- 추가 XP 보너스 (500 XP)
- 크루 이벤트 우선 참여

## 프로젝트 가시성 설정

프로젝트의 공개 범위를 설정하여 누가 볼 수 있는지 제어할 수 있습니다.

### PUBLIC (공개)

**누가 볼 수 있나요?**
- 모든 사람 (로그인하지 않은 방문자 포함)
- 검색 엔진에도 노출

**언제 사용하나요?**
- 완성된 프로젝트
- 포트폴리오로 활용
- 취업 준비용
- 오픈소스 프로젝트

**장점**:
- 높은 조회수
- 많은 피드백
- 네트워킹 기회

**단점**:
- 코드가 모두 공개됨
- 비판적 피드백 가능성

### PRIVATE (비공개)

**누가 볼 수 있나요?**
- 나만 볼 수 있음
- Owner만 접근 가능

**언제 사용하나요?**
- 진행 중인 프로젝트
- 실험적인 프로젝트
- 개인 학습용
- 민감한 정보 포함

**장점**:
- 완전한 프라이버시
- 실수해도 괜찮음
- 마음 편하게 개발

**단점**:
- XP 획득 제한
- 피드백 받기 어려움
- 노출 기회 없음

### TEAM (팀 전용)

**누가 볼 수 있나요?**
- 팀원으로 추가된 크루원만
- 로그인 필수

**언제 사용하나요?**
- 팀 프로젝트 진행 중
- 완성 전 팀 내부 검토
- 멘토-멘티 프로젝트

**장점**:
- 팀 내에서만 공유
- 외부 노출 없음
- 협업에 최적화

**단점**:
- 제한된 노출
- 쇼케이스 등록 불가

### 가시성 변경

프로젝트를 진행하면서 가시성을 변경할 수 있습니다.

**일반적인 흐름**:
```
PRIVATE (개발 시작)
    ↓
TEAM (팀 협업)
    ↓
PUBLIC (완성 후 공개)
```

**변경 방법**:
1. 프로젝트 편집 페이지
2. 가시성 설정 선택
3. 저장

**주의**: PUBLIC → PRIVATE 변경 시 이미 받은 조회수, 좋아요는 유지됩니다.

## 프로젝트 활용 팁

### 포트폴리오로 활용하기

**프로젝트 선택**:
- 3-5개의 대표 프로젝트 선정
- 다양한 기술 스택 사용
- 완성도 높은 프로젝트 우선

**프로젝트 설명 작성**:
```markdown
## 프로젝트 배경
왜 이 프로젝트를 시작했는지 설명

## 해결한 문제
어떤 문제를 해결했는지

## 기술적 도전
어떤 기술적 어려움을 극복했는지

## 성과 및 학습
무엇을 배웠는지, 어떤 성과를 냈는지
```

**비주얼 강화**:
- 스크린샷 여러 장 포함
- GIF로 동작 과정 보여주기
- 아키텍처 다이어그램 추가

### 좋아요 늘리기

**고품질 프로젝트**:
- 실제로 동작하는 배포 링크
- 깔끔한 UI/UX
- 버그 없는 완성도

**커뮤니티 활동**:
- 프로젝트 완성 소식을 커뮤니티에 공유
- 다른 크루원 프로젝트에 좋아요 및 피드백
- 기술 블로그 글 작성

**업데이트 지속**:
- 주기적으로 기능 추가
- 버전 업데이트 공지
- 사용자 피드백 반영

### GitHub 연동 최대 활용

**README.md 작성**:
- 프로젝트 소개
- 설치 및 실행 방법
- 기술 스택
- 스크린샷
- 라이선스 정보

**커밋 메시지**:
- 명확한 커밋 메시지 작성
- Conventional Commits 형식 사용
- 예시: `feat: 사용자 로그인 기능 추가`

**브랜치 전략**:
- main: 안정 버전
- develop: 개발 중
- feature/*: 새 기능 개발

## 자주 묻는 질문

### Q1. 프로젝트를 몇 개까지 등록할 수 있나요?

**A**: 제한 없습니다. 원하는 만큼 프로젝트를 등록할 수 있습니다. 다만, 포트폴리오로는 3-5개의 대표 프로젝트를 선별하는 것을 추천합니다.

### Q2. 완성되지 않은 프로젝트도 등록할 수 있나요?

**A**: 가능합니다. PRIVATE 또는 TEAM 가시성으로 등록하고, 완성 후 PUBLIC으로 변경하세요. 진행 중인 프로젝트를 등록하면 진도를 추적하고 동기 부여를 받을 수 있습니다.

### Q3. 프로젝트를 수정하면 XP를 계속 받나요?

**A**: 프로젝트 수정 XP는 주 1회만 지급됩니다. 남용을 방지하기 위한 제한입니다.

### Q4. 팀 프로젝트는 어떻게 XP가 분배되나요?

**A**: 팀 프로젝트의 XP는 모든 팀원에게 동일하게 지급됩니다. Owner, Admin, Member 모두 같은 XP를 받습니다.

### Q5. 쇼케이스 신청이 거부되면 어떻게 하나요?

**A**: 거부 사유를 확인하고 프로젝트를 보완한 후 재신청할 수 있습니다. 재신청 횟수에는 제한이 없습니다.

### Q6. 프로젝트를 삭제하면 XP도 회수되나요?

**A**: 이미 받은 XP는 회수되지 않습니다. 하지만 프로젝트 삭제는 통계에 영향을 미치므로 신중하게 결정하세요.

### Q7. GitHub private 저장소도 연동할 수 있나요?

**A**: 현재는 공개 저장소만 연동 가능합니다. 향후 GitHub App 인증을 통해 private 저장소도 지원할 예정입니다.
