## 2. Primary Persona 1: Minji - The Curious Beginner

### 2.1 Demographics

| Attribute | Details |
|-----------|---------|
| **Name** | Park Minji (박민지) |
| **Age** | 19 |
| **Gender** | Female |
| **Location** | Iksan, Jeollabuk-do |
| **Education** | Wonkwang University, Computer Science, Freshman |
| **Income** | ₩500,000/month (part-time job + allowance) |
| **Living** | Dormitory |

### 2.2 Background

Minji just started her CS degree with minimal programming experience. She chose CS because of interest in technology but feels overwhelmed by the technical coursework. She's eager to learn but struggles with abstract concepts without practical application.

**Tech Proficiency**: 2/10
- Can use Microsoft Office, social media
- No programming experience
- Intimidated by command line and technical tools

### 2.3 Goals

**Primary Goals**:
- ✅ Learn programming fundamentals (Python, JavaScript)
- ✅ Build her first real project
- ✅ Understand what CS career paths are available
- ✅ Make friends in the CS community

**Secondary Goals**:
- Pass her programming courses with good grades
- Build confidence in her technical abilities
- Explore different areas of CS to find her interest

### 2.4 Pain Points

1. **Overwhelming Complexity**:
   - "I don't know where to start"
   - Too many languages, frameworks, tools
   - Abstract concepts without practical context

2. **Expensive Resources**:
   - Online courses cost ₩50,000-₩200,000
   - Can't afford multiple course subscriptions

3. **Setup Difficulties**:
   - Struggled for 3 hours setting up Python
   - Different instructions for Windows/Mac
   - "It works on my professor's computer but not mine"

4. **Lack of Community**:
   - Feels alone in her learning journey
   - Too embarrassed to ask "basic" questions in class
   - No peers at her level to study with

5. **Theory-Heavy Curriculum**:
   - Lectures focus on theory, not practice
   - Assignments are disconnected from real-world applications

### 2.5 Motivations

- **Intrinsic**: Curiosity about technology, desire to create
- **Extrinsic**: Good grades, future job prospects, parental expectations
- **Social**: Belonging to CS community, making friends

### 2.6 Behaviors

**Daily Routine**:
- 6:00 AM: Wake up
- 7:30 AM - 12:00 PM: Classes
- 12:00 PM - 1:00 PM: Lunch with friends
- 1:00 PM - 3:00 PM: Study/homework
- 3:00 PM - 6:00 PM: Part-time job (cafe)
- 6:00 PM - 7:00 PM: Dinner
- 7:00 PM - 10:00 PM: Study, YouTube coding tutorials
- 10:00 PM - 11:00 PM: Relax, social media
- 11:00 PM: Sleep

**Learning Style**:
- Visual learner (prefers video tutorials)
- Learns by doing (hands-on projects)
- Needs frequent encouragement and positive feedback
- Short attention span (15-20 minute sessions)

**Technology Usage**:
- Smartphone: 5-6 hours/day (Instagram, YouTube, KakaoTalk)
- Laptop: 3-4 hours/day (studying, assignments)
- Tablet: Occasionally for reading

**Information Sources**:
- YouTube tutorials (free)
- University lecture notes
- Asks seniors for advice
- Google search for quick answers

### 2.7 User Journey

**Stage 1: Discovery** (Week 1)
- Sees WKU Software Crew poster on campus
- Curious about "free coding platform"
- Visits website on phone during break

**Stage 2: Sign-up** (Week 1)
- Creates account with university email
- Quick signup process (< 2 minutes)
- Receives welcome email

**Stage 3: Onboarding** (Week 1-2)
- Watches 5-minute intro video
- Completes "Hello World" tutorial
- Gets first badge → feels accomplished
- Joins "Beginner Study Group"

**Stage 4: Engagement** (Week 3-8)
- Follows structured Junior Class curriculum
- Watches video lectures during commute
- Works on exercises in cloud IDE (no setup!)
- Posts first question in Q&A forum
- Gets helpful answer within 1 hour → feels supported
- Completes first small project → proud moment
- Shares project link with friends

**Stage 5: Habit Formation** (Month 3-6)
- Logs in 4-5 times/week
- Part of active study group (5 members)
- Contributes to community discussions
- Earned 10+ badges
- Working on portfolio project

**Stage 6: Conversion Decision** (Month 6)
- Completed Junior Class
- Wants to access Senior Class content
- Considers Premium subscription (₩9,900/mo)
- Sees value: "Less than 2 cafe drinks per month"
- Subscribes with 7-day free trial

### 2.8 How WKU Software Crew Helps

| Pain Point | Solution |
|------------|----------|
| **Overwhelming** | Structured Junior Class curriculum with clear progression |
| **Expensive** | Free Junior tier, affordable Premium (₩9,900/mo) |
| **Setup Issues** | Cloud IDE - works in browser, no installation |
| **Isolation** | Study groups, Q&A forum, beginner-friendly community |
| **Theory-Heavy** | Project-based learning with real applications |

### 2.9 Success Metrics

- Completes Junior Class within 6 months
- Posts 5+ questions/answers in community
- Builds 3+ projects in portfolio
- Converts to Premium after 6 months
- Refers 2+ friends to platform

### 2.10 Quote

> "I finally understand what programming is actually for. Building my first web app was the proudest moment of my freshman year. And I didn't have to spend hours installing stuff - I just opened the browser and started coding!"

## 15. Accessibility Considerations by Persona

### 15.1 Special Needs

**Visual Impairments**:
- Screen reader compatibility (WCAG 2.1 AA)
- High contrast mode
- Adjustable font sizes
- Keyboard navigation

**Cognitive Considerations** (especially for Minji):
- Simple language, avoid jargon
- Clear visual hierarchy
- Consistent navigation
- Undo/redo functionality

**Time Constraints** (Yuna - part-time learner):
- Save progress automatically
- Resume where left off
- Offline mode for video lectures
- Mobile-friendly for learning on-the-go
