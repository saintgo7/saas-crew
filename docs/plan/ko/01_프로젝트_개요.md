# WKU 소프트웨어 Crew Project - 프로젝트 개요

## 1. 프로젝트 정의

### 1.1 프로젝트 명칭
- **정식 명칭**: WKU Software Crew (원광대학교 소프트웨어 크루)
- **약칭**: WKU CREW
- **영문 명칭**: Wonkwang University Software Crew Platform

### 1.2 프로젝트 비전
"모든 학생이 소프트웨어 개발 역량을 키우고, 자유롭게 창업할 수 있는 혁신적인 교육 생태계 구축"

### 1.3 프로젝트 미션
- 단계별 체계적인 프로그래밍 교육 제공
- 실무 중심의 협업 및 프로젝트 경험 축적
- 스타트업 창업을 위한 기술적 기반 마련
- 학생-교수-산업 간 지식 공유 플랫폼 구축

## 2. 프로젝트 배경

### 2.1 현황 분석
- **교육 환경**: 전통적인 강의식 교육의 한계
- **학생 니즈**: 실무 경험과 포트폴리오 구축 욕구 증가
- **산업 요구**: 즉시 투입 가능한 실무형 개발자 수요 급증
- **창업 트렌드**: 대학생 창업 활성화 및 정부 지원 확대

### 2.2 문제점 및 기회
**문제점**
- 이론과 실무의 괴리
- 단계적 학습 경로 부재
- 협업 경험 부족
- 창업 준비를 위한 기술 인프라 미비

**기회 요인**
- 온라인 교육 플랫폼 시장 성장
- 코딩 교육 수요 폭발적 증가
- 정부의 대학 혁신 지원 정책
- 클라우드 기반 개발 환경 접근성 향상

## 3. 프로젝트 목표

### 3.1 정량적 목표 (1년차)
- 플랫폼 등록 학생: 500명
- Junior 클래스 완료율: 70%
- Senior 클래스 진입률: 40%
- Master 클래스 진입률: 10%
- 창업 프로젝트 론칭: 5개 이상
- 플랫폼 활성 사용자(MAU): 300명
- 월간 코드 커밋 수: 1,000건 이상

### 3.2 정성적 목표
- 학생 주도적 학습 문화 확산
- 실무 중심 프로젝트 경험 제공
- 지속 가능한 학습 커뮤니티 형성
- 교수-학생 간 원활한 지식 공유
- 산학 협력 강화

## 4. 핵심 가치 제안 (Value Proposition)

### 4.1 학생 대상
- **체계적 학습 경로**: Junior → Senior → Master 단계별 성장
- **실전 프로젝트**: 실제 배포 가능한 서비스 개발 경험
- **포트폴리오 구축**: GitHub 연동 및 자동화된 포트폴리오 생성
- **창업 지원**: 아이디어부터 배포까지 All-in-One 지원
- **커뮤니티**: 동료 학습자 및 멘토와의 네트워킹

### 4.2 교수자 대상
- **통합 교육 관리**: 학생 진도 및 성과 실시간 모니터링
- **콘텐츠 공유**: 강의자료, 과제, 프로젝트 템플릿 공유
- **자동화된 평가**: 코드 제출 및 자동 채점 시스템
- **데이터 기반 인사이트**: 학습 분석 대시보드

### 4.3 대학 대상
- **브랜드 가치**: 혁신적인 교육 모델 구축
- **취업률 향상**: 실무 역량을 갖춘 졸업생 배출
- **창업 생태계**: 대학 내 스타트업 육성 플랫폼
- **산학 협력**: 기업과의 프로젝트 연계 기회

## 5. 핵심 기능 개요

### 5.1 학습 관리 시스템 (LMS)
- 단계별 커리큘럼 (Junior/Senior/Master)
- 인터랙티브 코딩 튜토리얼
- 영상 강의 및 문서 자료
- 실습 과제 및 프로젝트

### 5.2 협업 개발 환경
- 클라우드 기반 IDE (VS Code 웹 버전)
- 실시간 코드 협업 (Live Share)
- Git/GitHub 통합
- 코드 리뷰 시스템

### 5.3 프로젝트 관리
- 프로젝트 생성 및 팀 구성
- 이슈 트래킹 (Kanban 보드)
- 버전 관리
- 배포 자동화 (CI/CD)

### 5.4 커뮤니티 & 소셜
- 질의응답 포럼
- 스터디 그룹
- 멘토링 매칭
- 포트폴리오 쇼케이스

### 5.5 창업 지원
- 아이디어 등록 및 검증
- 팀 빌딩
- 프로토타입 개발 지원
- 투자 연계 (데모데이)

## 6. 타겟 사용자

### 6.1 Primary Users
**초보 학습자 (Junior Class)**
- 연령: 18-22세 (1-2학년)
- 프로그래밍 경험: 0-1년
- 목표: 프로그래밍 기초 학습 및 첫 프로젝트 완성

**중급 개발자 (Senior Class)**
- 연령: 20-24세 (2-4학년)
- 프로그래밍 경험: 1-3년
- 목표: 풀스택 개발 능력 및 팀 프로젝트 경험

**고급 개발자 (Master Class)**
- 연령: 22-26세 (3-4학년, 대학원생)
- 프로그래밍 경험: 3년 이상
- 목표: 아키텍처 설계, 오픈소스 기여, 창업

### 6.2 Secondary Users
**교수 및 강사**
- 교육 콘텐츠 제작자
- 멘토 및 코드 리뷰어
- 학생 진도 관리자

**기업 파트너**
- 산학 프로젝트 제공
- 인턴/채용 연계
- 기술 멘토링

## 7. 성공 지표 (KPI)

### 7.1 사용자 지표
- 신규 가입자 수 (월별)
- 활성 사용자 수 (DAU/MAU)
- 사용자 유지율 (Retention Rate)
- 클래스 레벨 전환율

### 7.2 학습 지표
- 코스 완료율
- 평균 학습 시간
- 과제 제출률
- 코드 품질 점수

### 7.3 협업 지표
- 프로젝트 생성 수
- 팀 활동 지표 (커밋, 리뷰 수)
- 커뮤니티 참여도 (포럼 활동)

### 7.4 비즈니스 지표
- 월간 반복 수익 (MRR)
- 고객 생애 가치 (LTV)
- 고객 획득 비용 (CAC)
- 창업 프로젝트 론칭 수

## 8. 프로젝트 범위

### 8.1 포함 범위 (In Scope)
- 웹 기반 학습 플랫폼
- 클라우드 IDE 통합
- Junior/Senior/Master 3단계 커리큘럼
- 프로젝트 협업 도구
- 커뮤니티 기능
- 기본 분석 대시보드

### 8.2 제외 범위 (Out of Scope)
- 모바일 네이티브 앱 (1차 버전)
- 실시간 화상 강의 (외부 도구 연동)
- 결제 시스템 (초기에는 무료 운영)
- AI 기반 코드 자동 완성 (2차 버전)

### 8.3 향후 확장 가능성
- 다른 대학으로 확장
- 기업 교육 프로그램
- AI 튜터 및 코드 리뷰
- 자격증 및 인증 프로그램

## 9. 주요 이해관계자

### 9.1 내부 이해관계자
- **원광대학교 소프트웨어학과**: 프로젝트 주관
- **학생 개발팀**: 플랫폼 개발 및 유지보수
- **교수진**: 커리큘럼 설계 및 멘토링
- **대학 행정**: 예산 및 행정 지원

### 9.2 외부 이해관계자
- **학생 사용자**: 주요 고객
- **기업 파트너**: 프로젝트 및 채용 연계
- **정부 기관**: 창업 지원 프로그램
- **오픈소스 커뮤니티**: 기술 생태계

## 10. 프로젝트 제약사항

### 10.1 기술적 제약
- 초기 예산 한계로 인한 인프라 제약
- 레거시 시스템과의 통합 필요성
- 보안 및 개인정보 보호 요구사항

### 10.2 조직적 제약
- 학기 단위 운영 주기
- 학생 개발 인력의 유동성
- 교수진의 시간적 제약

### 10.3 시장 제약
- 경쟁 플랫폼 존재 (Udemy, Coursera 등)
- 무료 오픈소스 도구 다수 존재
- 사용자 습관 변경의 어려움

## 11. 리스크 요인

### 11.1 기술적 리스크
- 클라우드 인프라 비용 초과
- 확장성 문제
- 보안 취약점

### 11.2 운영적 리스크
- 사용자 채택률 저조
- 콘텐츠 품질 관리
- 지속적인 업데이트 부담

### 11.3 재정적 리스크
- 수익 모델 불확실성
- 유지보수 비용
- 초기 투자 회수 기간

## 12. 프로젝트 일정 (High-Level)

### Phase 1: 기획 및 설계 (1-2개월)
- 요구사항 정의
- 시스템 아키텍처 설계
- UI/UX 디자인

### Phase 2: MVP 개발 (3-4개월)
- 핵심 기능 개발
- Junior 클래스 커리큘럼
- 기본 협업 기능

### Phase 3: 베타 테스트 (1개월)
- 파일럿 그룹 운영
- 피드백 수집 및 개선

### Phase 4: 정식 런칭 (1개월)
- 전체 학생 대상 오픈
- 마케팅 캠페인
- 모니터링 및 최적화

### Phase 5: 확장 및 고도화 (지속)
- Senior/Master 클래스 추가
- 고급 기능 개발
- 타 대학 확장
