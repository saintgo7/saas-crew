## FR-1. 사용자 관리 (User Management)

### FR-1.1 회원가입 및 로그인

#### FR-1.1.1 이메일 회원가입 [P0]
**설명**: 사용자가 이메일로 회원가입

**사용자 스토리**:
- As a 학생, I want to 이메일로 회원가입하고 싶다, so that 플랫폼을 사용할 수 있다

**입력**:
- 이름 (필수)
- 이메일 (필수, 학교 이메일 권장)
- 비밀번호 (필수, 최소 8자, 영문+숫자+특수문자)
- 학과 (필수)
- 학년 (필수)
- 약관 동의 (필수)

**출력**:
- 이메일 인증 메일 발송
- 계정 생성 (인증 대기 상태)

**비즈니스 규칙**:
- BR-1: 이메일은 중복 불가
- BR-2: 학교 이메일(@wku.ac.kr)은 자동 인증
- BR-3: 비밀번호는 bcrypt 암호화

**검증 규칙**:
- VR-1: 이메일 형식 검증 (RFC 5322)
- VR-2: 비밀번호 강도 검증 (zxcvbn)
- VR-3: XSS 방지 (입력 sanitize)

**UI 요구사항**:
- 간결한 1페이지 폼
- 실시간 입력 검증
- 비밀번호 강도 인디케이터
- 소셜 로그인 버튼 (하단)

**성능 요구사항**:
- 회원가입 처리: < 2초
- 이메일 발송: < 5초

---

#### FR-1.1.2 소셜 로그인 [P0]
**설명**: GitHub, Google 계정으로 로그인

**지원 제공자**:
- GitHub (OAuth 2.0)
- Google (OAuth 2.0)
- (향후) 카카오, 네이버

**프로세스**:
1. 사용자가 "GitHub로 로그인" 클릭
2. GitHub 인증 페이지로 리다이렉트
3. 사용자 승인
4. Callback URL로 돌아옴
5. Access Token 획득
6. 사용자 정보 조회
7. 계정 생성 or 로그인

**자동 연동**:
- GitHub 연동 시: 레포지토리 자동 연결
- Google 연동 시: 캘린더 동기화 (선택)

---

#### FR-1.1.3 이메일 인증 [P0]
**설명**: 회원가입 후 이메일 인증

**프로세스**:
1. 회원가입 시 인증 메일 발송
2. 사용자가 링크 클릭
3. 토큰 검증 (24시간 유효)
4. 계정 활성화

**재발송 기능**:
- 인증 메일 재발송 (5분 쿨타임)

---

#### FR-1.1.4 로그인/로그아웃 [P0]
**설명**: 이메일 또는 소셜 로그인

**로그인 옵션**:
- 이메일 + 비밀번호
- GitHub OAuth
- Google OAuth
- "로그인 상태 유지" 체크박스

**세션 관리**:
- JWT 토큰 기반
- Access Token: 1시간 유효
- Refresh Token: 30일 유효
- 자동 로그아웃: 30일 미사용 시

**보안**:
- 비밀번호 5회 실패 시 15분 잠금
- IP 기반 이상 로그인 감지

---

#### FR-1.1.5 비밀번호 재설정 [P0]
**설명**: 비밀번호 분실 시 재설정

**프로세스**:
1. "비밀번호 찾기" 클릭
2. 이메일 입력
3. 재설정 링크 발송 (1시간 유효)
4. 새 비밀번호 입력
5. 비밀번호 변경 완료
