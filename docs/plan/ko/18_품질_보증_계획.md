# WKU 소프트웨어 Crew Project - 품질 보증 계획

## 1. 품질 보증 개요

### 1.1 품질 목표
- 버그 발생률: < 1% (주요 기능)
- 사용자 만족도: > 4.5/5.0
- 시스템 가용성: 99.9%
- 응답 시간: < 500ms (p95)

---

## 2. 테스트 전략

### 2.1 테스트 피라미드

```
           E2E Tests (10%)
              /\
             /  \
        Integration Tests (30%)
           /      \
          /        \
     Unit Tests (60%)
```

### 2.2 테스트 유형

#### 단위 테스트 (Unit Tests)
- **도구**: Vitest (프론트), Jest (백엔드)
- **커버리지 목표**: 80% 이상
- **실행**: CI/CD 파이프라인 자동 실행

#### 통합 테스트 (Integration Tests)
- **도구**: Supertest (API), React Testing Library
- **범위**: API 엔드포인트, 컴포넌트 통합

#### E2E 테스트 (End-to-End Tests)
- **도구**: Playwright
- **범위**: 핵심 사용자 플로우
  - 회원가입 → 로그인 → 첫 강의 수강
  - 프로젝트 생성 → 코드 작성 → 배포

#### 성능 테스트 (Performance Tests)
- **도구**: k6, Lighthouse
- **지표**: 응답 시간, 처리량, 메모리 사용

#### 보안 테스트 (Security Tests)
- **도구**: OWASP ZAP, Snyk
- **빈도**: 주간

## 3. 코드 품질

### 3.1 코드 리뷰

**필수 사항**:
- 모든 PR은 최소 1명 승인 필요
- Self-review 금지
- 체크리스트 준수

**리뷰 체크리스트**:
- [ ] 코드 스타일 준수 (Linter 통과)
- [ ] 테스트 작성 및 통과
- [ ] 보안 취약점 없음
- [ ] 성능 이슈 없음
- [ ] 문서 업데이트

### 3.2 정적 분석

- **ESLint**: JavaScript/TypeScript
- **Prettier**: 코드 포매팅
- **SonarQube**: 코드 품질 분석
