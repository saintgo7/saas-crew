# WKU 소프트웨어 Crew Project - 데이터 분석 전략

## 1. 데이터 분석 목표

### 1.1 핵심 목표
- 사용자 행동 이해
- 제품 개선 인사이트 도출
- 비즈니스 의사결정 지원
- 개인화 및 추천 시스템

## 2. 데이터 수집

### 2.1 수집 데이터

#### 사용자 데이터
- 회원 정보 (이름, 이메일, 학과, 학년)
- 프로필 (프로필 사진, 자기소개, 스킬)

#### 행동 데이터
- 페이지 방문 (URL, 시간, 체류 시간)
- 클릭 이벤트 (버튼, 링크)
- 스크롤 깊이

#### 학습 데이터
- 코스 등록, 진행률, 완료
- 챕터별 학습 시간
- 과제 제출 및 점수
- 동영상 시청 (재생, 일시정지, 이탈 시점)

#### 프로젝트 데이터
- 프로젝트 생성, 수정, 삭제
- 파일 편집 빈도
- 커밋 횟수
- 배포 횟수

#### 커뮤니티 데이터
- 게시글 작성, 조회, 좋아요
- 댓글 작성
- 검색 키워드

### 2.2 수집 도구

- **Google Analytics 4**: 웹 분석
- **Mixpanel**: 사용자 행동 분석
- **Custom Events**: 플랫폼 내 이벤트 (Amplitude)

---

## 3. 데이터 분석 프레임워크

### 3.1 AARRR Funnel 분석

#### Acquisition (획득)
- 신규 가입자 수 (일/주/월)
- 유입 채널 (검색, SNS, 추천 등)
- CAC (고객 획득 비용)

#### Activation (활성화)
- 첫 강의 수강률
- 온보딩 완료율
- Time to First Value

#### Retention (리텐션)
- DAU/MAU Ratio
- Cohort Retention (코호트별 유지율)
- Churn Rate (이탈률)

#### Revenue (수익)
- 유료 전환율
- ARPU (사용자당 평균 수익)
- LTV (고객 생애 가치)

#### Referral (추천)
- 추천 프로그램 참여율
- K-Factor (바이럴 계수)
- 추천으로 유입된 사용자 비율

### 3.2 코호트 분석

**목적**: 가입 시기별 사용자 행동 비교

**예시**:
- 1월 가입 코호트: Week 1 Retention 60%
- 2월 가입 코호트: Week 1 Retention 70%
→ 온보딩 개선 효과 확인

### 3.3 Funnel 분석

**회원가입 퍼널**:
1. 랜딩 페이지 방문: 1,000명
2. 회원가입 시작: 600명 (60%)
3. 이메일 입력: 500명 (83%)
4. 정보 입력: 450명 (90%)
5. 가입 완료: 400명 (89%)

→ 이탈이 많은 단계 개선

## 5. A/B 테스트

### 5.1 테스트 프로세스

1. **가설 수립**: "CTA 버튼 색을 파란색으로 바꾸면 클릭률이 20% 증가할 것이다"
2. **실험 설계**: A그룹 (기존), B그룹 (변경)
3. **실행**: 2주간 실행
4. **분석**: 통계적 유의성 확인 (p < 0.05)
5. **결정**: 승자 선택 또는 기각

### 5.2 주요 테스트 항목

- 회원가입 플로우
- CTA 버튼 (색상, 문구, 위치)
- 가격 표시 (₩9,900 vs ₩10,000)
- 이메일 제목
