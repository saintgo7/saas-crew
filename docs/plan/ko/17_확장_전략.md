# WKU 소프트웨어 Crew Project - 확장 전략

## 1. 확장 로드맵

### 1.1 지역 확장

**1년차**: 원광대학교 (익산/전주)
**2년차**: 전라북도 (전북대, 군산대, 전주대)
**3년차**: 호남권 (전남대, 조선대)
**4년차**: 전국 (서울, 수도권, 부산)
**5년차**: 해외 (동남아시아)

### 1.2 제품 확장

**1년차**: Junior 클래스 (입문)
**2년차**: Senior 클래스 (중급)
**3년차**: Master 클래스 (고급 + 창업)
**4년차**: 기업 교육 버전
**5년차**: 고등학생 버전

---

## 2. 수평 확장 (Horizontal Scaling)

### 2.1 타 대학 확장

#### 진입 전략
1. **선정 기준**
   - 소프트웨어 관련 학과 존재
   - 학생 수 1,000명 이상
   - 학교 혁신 의지

2. **진입 방법**
   - 파일럿 프로그램 (3개월 무료)
   - 교수 네트워킹
   - 학생 대사 프로그램

3. **현지화**
   - 학교별 브랜딩
   - 학사 시스템 연동
   - 지역 특화 콘텐츠

#### 확장 목표

| 연도 | 대학 수 | 누적 학생 수 |
|------|---------|--------------|
| 1년차 | 1개 | 500명 |
| 2년차 | 3개 | 5,000명 |
| 3년차 | 10개 | 30,000명 |
| 5년차 | 50개 | 200,000명 |

### 2.2 세그먼트 확장

#### 일반 성인 (2년차)
- **타겟**: 직장인, 취업 준비생
- **니즈**: 직무 전환, 스킬업
- **차별화**: 야간/주말 과정, 단기 집중

#### 기업 교육 (3년차)
- **타겟**: IT 기업, 대기업
- **니즈**: 직원 교육, 온보딩
- **모델**: B2B SaaS (연 ₩50M/기업)

#### 고등학생 (4년차)
- **타겟**: 정보 과목 학생
- **니즈**: 코딩 교육, 대입 준비
- **차별화**: 게이미피케이션, 간결한 UI

---

## 3. 수직 확장 (Vertical Scaling)

### 3.1 상위 가치사슬

#### 취업 플랫폼
- 포트폴리오 기반 매칭
- 기업 연계
- 이력서 자동 생성

#### 프리랜서 매칭
- 학생 개발자 ↔ 프로젝트
- 단기 프로젝트 중개
- 수수료: 10-15%

#### 컨설팅 서비스
- 기업 기술 자문
- 프로젝트 아웃소싱

### 3.2 하위 가치사슬

#### 코딩 부트캠프 (오프라인)
- 방학 집중 과정 (2주)
- 지역별 개최
- 참가비: ₩50만원

#### 초등학생 코딩 교육
- 스크래치, 엔트리
- 여름/겨울 캠프

## 4. 제품 확장

### 4.1 신규 기능

#### AI 기능 (2년차)
- AI 코드 리뷰
- AI 튜터 (ChatGPT 연동)
- 학습 경로 추천

#### 모바일 앱 (2년차)
- 학습 진도 확인
- 알림
- 간단한 코딩 연습

#### VR/AR (5년차)
- 가상 강의실
- 실감나는 협업

### 4.2 콘텐츠 확장

#### 언어 확장
- 현재: JavaScript, Python
- 추가: Java, C++, Go, Rust

#### 분야 확장
- 웹 개발 → 앱 개발
- → 데이터 사이언스
- → AI/ML
- → 블록체인

## 6. 확장 리스크 관리

### 6.1 주요 리스크

**리스크 1: 품질 저하**
- 빠른 확장 → 품질 관리 어려움
- 대응: 품질 기준 엄격히 유지

**리스크 2: 문화적 차이**
- 지역/국가별 차이
- 대응: 현지화 팀 구성

**리스크 3: 경쟁 심화**
- 확장 지역에 경쟁사 존재
- 대응: 차별화 강화

### 6.2 확장 원칙

1. **검증 후 확장**: 파일럿 필수
2. **점진적 확장**: 한 번에 많은 지역 X
3. **핵심 강점 유지**: 차별화 포인트 보존
