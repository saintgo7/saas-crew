## 6. 개발 방법론

### 6.1 애자일 스크럼

#### 스프린트
- 기간: 2주
- 스프린트 플래닝: 월요일 오전
- 데일리 스탠드업: 매일 오전 10시 (15분)
- 스프린트 리뷰: 격주 금요일
- 회고: 스프린트 종료 후

#### 스프린트 플래닝
1. 백로그 우선순위 확인
2. 스프린트 목표 설정
3. 태스크 분배
4. 추정 (Story Points)

#### 데일리 스탠드업
- 어제 한 일
- 오늘 할 일
- 장애물

### 6.2 코드 리뷰

#### 프로세스
1. 개발자: 기능 구현 후 PR 생성
2. 리뷰어: 24시간 내 리뷰
3. 개발자: 피드백 반영
4. 리뷰어: 승인
5. Merge

#### 리뷰 기준
- 코드 스타일 (ESLint, Prettier)
- 테스트 커버리지 (최소 80%)
- 성능
- 보안

### 6.3 브랜치 전략 (Git Flow)

```
main (프로덕션)
  ↓
develop (개발)
  ↓
feature/* (기능 개발)
  ↓
release/* (릴리스 준비)
  ↓
hotfix/* (긴급 수정)
```
