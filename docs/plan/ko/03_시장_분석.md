# WKU 소프트웨어 Crew Project - 시장 분석

## 1. 시장 개요

### 1.1 글로벌 시장 규모

#### 온라인 교육 시장
- **2025년 시장 규모**: $325 billion (약 421조원)
- **2030년 예상 규모**: $1 trillion (약 1,295조원)
- **연평균 성장률 (CAGR)**: 20.5%

#### 코딩 교육 시장
- **2025년 시장 규모**: $41 billion (약 53조원)
- **2030년 예상 규모**: $102 billion (약 132조원)
- **연평균 성장률 (CAGR)**: 20.0%

#### 개발자 도구 시장
- **2025년 시장 규모**: $29 billion (약 38조원)
- **2030년 예상 규모**: $61 billion (약 79조원)
- **연평균 성장률 (CAGR)**: 16.0%

### 1.2 국내 시장 규모

- **2030년 예상 규모**: ₩2조

- **국내 대학생 수**: 약 190만명
- **SW 관련 학과 재학생**: 약 25만명
- **온라인 학습 플랫폼 사용 비율**: 78%

### 1.3 타겟 시장 규모 (TAM-SAM-SOM)

#### TAM (Total Addressable Market)
**국내 전체 대학생 + 성인 코딩 학습자**
- 대학생: 1,900,000명
- 성인 학습자: 500,000명
- **총 TAM**: 2,400,000명
- **시장 규모**: 2,400,000명 × ₩118,800/년 = ₩285억/년

#### SAM (Serviceable Available Market)
**SW 관련 학과 + 코딩 관심 학생**
- SW 관련 학과: 250,000명
- 코딩 부트캠프 수강생: 50,000명
- **총 SAM**: 300,000명
- **시장 규모**: 300,000명 × ₩118,800/년 = ₩356억/년

#### SOM (Serviceable Obtainable Market)
**초기 3년 목표 시장**
- 1년차: 500명 (원광대)
- 2년차: 5,000명 (전라북도 대학)
- 3년차: 30,000명 (전국 주요 대학)
- **3년차 시장 규모**: 30,000명 × ₩118,800/년 = ₩36억/년

**시장 침투율**: 10% (SOM/SAM)

## 2. 시장 트렌드

### 2.1 거시적 트렌드

#### 1. 디지털 전환 가속화
- **코로나19 이후 온라인 교육 정착**: 2020년 대비 300% 증가
- **하이브리드 교육 모델 확산**: 온라인 70% + 오프라인 30%
- **메타버스 교육 실험**: VR/AR 코딩 교육 시범 운영

#### 2. 개발자 수요 폭증
- **국내 SW 개발자 부족**: 약 10만명 (2025년 기준)
- **평균 연봉 상승**: 신입 개발자 평균 4,000만원 (전년 대비 15% 상승)
- **비전공자 개발자 증가**: 전체 개발자의 35%

#### 3. 스타트업 생태계 성장
- **대학생 창업 지원 확대**: 정부 예산 5,000억원 (2025년)
- **유니콘 기업 배출**: 2025년 국내 유니콘 20개 (2020년 대비 2배)
- **VC 투자 증가**: 스타트업 투자액 8조원 (2025년)

#### 4. AI 및 자동화
- **AI 코딩 어시스턴트**: GitHub Copilot 사용자 200만명 돌파
- **노코드/로우코드 플랫폼**: 시장 규모 $13 billion (2025년)
- **AI 기반 학습 개인화**: 학습 효율 30-40% 향상

#### 5. 평생 학습 문화
- **직무 전환 교육 수요**: 연간 50만명 (국내)
- **마이크로 러닝**: 평균 학습 세션 15분 이하
- **학위 대신 스킬**: 기업의 70%가 학위보다 실무 능력 중시

### 2.2 기술 트렌드

#### 1. 클라우드 네이티브
- **클라우드 IDE**: GitPod, CodeSandbox 등 급성장
- **브라우저 기반 개발**: 로컬 환경 설정 불필요
- **협업 도구 통합**: Notion, Slack, GitHub 등 통합 워크스페이스

#### 2. 실시간 협업
- **Pair Programming**: 원격 페어 프로그래밍 도구 인기
- **Live Share**: VS Code Live Share 월간 사용자 100만명
- **코드 리뷰 자동화**: AI 기반 코드 리뷰 도구

#### 3. 컨테이너화
- **Docker 표준화**: 개발 환경 일관성 확보
- **Kubernetes**: 프로덕션 배포 표준
- **마이크로서비스**: 모놀리식에서 마이크로서비스로 전환

#### 4. CI/CD
- **자동화 파이프라인**: GitHub Actions, GitLab CI 보편화
- **테스트 자동화**: 단위 테스트, 통합 테스트 자동화
- **무중단 배포**: Blue-Green, Canary 배포

### 2.3 교육 트렌드

#### 1. 프로젝트 기반 학습 (PBL)
- **실전 프로젝트 중심**: 이론 30% + 실습 70%
- **포트폴리오 중시**: 취업 시 포트폴리오 필수
- **해커톤 문화**: 대학 해커톤 연간 100회 이상 개최

#### 2. 마이크로 크레딧
- **모듈형 학습**: 과목 단위가 아닌 스킬 단위
- **디지털 배지**: 완료 증명 및 스킬 인증
- **누적 학점 시스템**: 다양한 플랫폼에서 학점 인정

#### 3. 게이미피케이션
- **레벨 시스템**: 학습 진도에 따른 레벨업
- **리더보드**: 경쟁 요소 도입으로 동기부여
- **보상 시스템**: 배지, 포인트, 랭킹

#### 4. 소셜 러닝
- **스터디 그룹**: 온라인 스터디 그룹 형성
- **멘토링**: 1:1 멘토링 매칭 플랫폼
- **커뮤니티 학습**: Q&A, 코드 리뷰 등 협업 학습

## 3. 고객 분석

### 3.1 고객 세그먼트

#### Junior Class (초급)
**인구통계학적 특성**
- 연령: 18-20세 (대학 1-2학년)
- 전공: 컴퓨터공학, 소프트웨어, 정보통신
- 프로그래밍 경험: 0-1년
- 비율: 전체 학생의 50%

**심리적 특성**
- 프로그래밍에 대한 호기심과 두려움 공존
- 진로 탐색 단계
- 또래 학습 선호
- 시각적 피드백 중시

**행동 특성**
- 주간 학습 시간: 5-10시간
- 선호 학습 방식: 동영상 강의 + 실습
- 커뮤니티 활동: 중간 (질문 위주)
- 포기율: 높음 (40-50%)

**니즈 및 동기**
- 프로그래밍 기초 이해
- 첫 프로젝트 완성 경험
- 학점 관리
- 진로 결정을 위한 탐색

**고객 가치 (LTV)**: ₩100,000 (1년)

#### Senior Class (중급)
**인구통계학적 특성**
- 연령: 20-23세 (대학 2-4학년)
- 프로그래밍 경험: 1-3년
- 프로젝트 경험: 1-3개
- 비율: 전체 학생의 35%

**심리적 특성**
- 실무 능력 향상 욕구
- 취업 준비 시작
- 협업 경험 추구
- 자기주도 학습 가능

**행동 특성**
- 주간 학습 시간: 10-20시간
- 선호 학습 방식: 프로젝트 기반 학습
- 커뮤니티 활동: 활발 (질문 + 답변)
- 지속률: 중간 (60-70%)

**니즈 및 동기**
- 풀스택 개발 능력
- 팀 프로젝트 경험
- 포트폴리오 구축
- 인턴십 기회

**고객 가치 (LTV)**: ₩300,000 (2년)

#### Master Class (고급)
**인구통계학적 특성**
- 연령: 22-26세 (대학 3-4학년, 대학원생)
- 프로그래밍 경험: 3년 이상
- 프로젝트 경험: 5개 이상
- 비율: 전체 학생의 15%

**심리적 특성**
- 전문가 지향
- 창업 또는 리드 개발자 목표
- 오픈소스 기여 관심
- 기술 리더십 추구

**행동 특성**
- 주간 학습 시간: 20-40시간
- 선호 학습 방식: 실전 프로젝트, 오픈소스
- 커뮤니티 활동: 매우 활발 (멘토 역할)
- 지속률: 높음 (80-90%)

**니즈 및 동기**
- 아키텍처 설계 능력
- 창업 기술 스택
- 투자 유치 준비
- 기술 리더십

**고객 가치 (LTV)**: ₩1,000,000 (3년)

### 3.2 고객 페르소나

#### 페르소나 1: 민지 (Junior)
- **나이**: 19세, 소프트웨어학과 1학년
- **목표**: 프로그래밍 기초를 배우고 첫 웹사이트 만들기
- **고충**: 혼자 공부하면 어디서 막히는지 모르겠어요
- **동기**: 개발자가 멋있어 보여서, 진로를 결정하고 싶어요
- **기술 수준**: HTML/CSS 조금, JavaScript 입문
- **선호 학습**: 단계별 튜토리얼, 동영상 강의
- **1일 학습 시간**: 1-2시간
- **기기**: 노트북 (Windows), 스마트폰

#### 페르소나 2: 준호 (Senior)
- **나이**: 22세, 컴퓨터공학과 3학년
- **목표**: 포트폴리오 3개 완성하고 대기업 인턴 지원
- **고충**: 혼자 프로젝트하면 동기부여가 안 돼요
- **동기**: 좋은 회사 취업, 실무 경험 쌓기
- **기술 수준**: React, Node.js, MySQL 중급
- **선호 학습**: 팀 프로젝트, 코드 리뷰
- **1일 학습 시간**: 3-4시간
- **기기**: 맥북, 아이패드

#### 페르소나 3: 수현 (Master)
- **나이**: 24세, 소프트웨어학과 4학년
- **목표**: AI 스타트업 창업, 시드 투자 유치
- **고충**: 기술 공동창업자를 찾기 어려워요
- **동기**: 내 아이디어를 현실로 만들고 싶어요
- **기술 수준**: 풀스택 (React, FastAPI, PostgreSQL, Docker)
- **선호 학습**: 실전 프로젝트, 멘토링, 네트워킹
- **1일 학습 시간**: 6-8시간
- **기기**: 맥북 프로, 클라우드 서버

### 3.3 고객 여정 맵 (Customer Journey Map)

#### Junior Class

**1단계: 인지 (Awareness)**
- 채널: 학과 공지, 선배 추천, SNS
- 생각: "프로그래밍을 배워볼까?"
- 감정: 호기심, 불안
- 액션: 플랫폼 방문, 둘러보기

**2단계: 고려 (Consideration)**
- 채널: 데모 영상, 무료 강의 체험
- 생각: "나도 할 수 있을까? 다른 플랫폼과 뭐가 다르지?"
- 감정: 흥미, 의심
- 액션: 무료 회원가입, 첫 강의 수강

**3단계: 전환 (Conversion)**
- 채널: 이메일, 푸시 알림
- 생각: "계속 배우려면 유료 결제해야 하나?"
- 감정: 고민, 기대
- 액션: Premium 결제 (또는 Free 유지)

**4단계: 사용 (Usage)**
- 채널: 플랫폼, 커뮤니티
- 생각: "재미있네! 다음 단계는 뭐지?"
- 감정: 성취감, 동기부여
- 액션: 강의 수강, 과제 제출, 프로젝트 진행

**5단계: 충성 (Loyalty)**
- 채널: 커뮤니티, 이벤트
- 생각: "이제 Senior로 레벨업해야지"
- 감정: 소속감, 자신감
- 액션: Senior Class 진입, 후배 멘토링, 플랫폼 홍보

#### Senior/Master Class 여정은 유사하나 더 깊은 참여 및 창업 활동 포함

## 4. 경쟁 환경

### 4.1 직접 경쟁자

#### 1. 국내 코딩 교육 플랫폼

**인프런 (Inflearn)**
- **강점**: 한국어 강의 풍부, 강사진 우수, 저렴한 가격
- **약점**: 프로젝트 협업 기능 없음, 커뮤니티 약함
- **시장 점유율**: 약 30%
- **타겟**: 일반 성인, 직장인
- **가격**: 강의당 ₩10,000-50,000

**프로그래머스 (Programmers)**
- **강점**: 코딩 테스트 특화, 기업 채용 연계
- **약점**: 학습 커리큘럼 부족, 초보자 진입 어려움
- **시장 점유율**: 약 20%
- **타겟**: 취업 준비생
- **가격**: 무료 (기업 B2B 수익)

**엘리스 (Elice)**
- **강점**: 클라우드 IDE 제공, 대학 파트너십
- **약점**: 콘텐츠 부족, 비싼 가격
- **시장 점유율**: 약 10%
- **타겟**: 대학, 기업 교육
- **가격**: B2B (대학/기업)

#### 2. 글로벌 플랫폼

**Udemy**
- **강점**: 방대한 강의, 저렴한 가격 (할인 시)
- **약점**: 한국어 강의 부족, 품질 편차 큼
- **시장 점유율**: 약 15% (국내)
- **타겟**: 전 세계 학습자
- **가격**: 강의당 ₩10,000-30,000

**Coursera**
- **강점**: 명문대 강의, 학위 인정
- **약점**: 비싼 가격, 한국어 지원 부족
- **시장 점유율**: 약 5% (국내)
- **타겟**: 글로벌 학습자
- **가격**: ₩49,000/월 (Plus)

### 4.2 간접 경쟁자

#### YouTube 무료 강의
- **강점**: 완전 무료, 다양한 콘텐츠
- **약점**: 체계적이지 않음, 실습 환경 없음
- **영향**: 초보자 유입 경쟁

#### 부트캠프
- **예**: 코드스테이츠, 패스트캠퍼스
- **강점**: 집중 교육, 취업 보장
- **약점**: 비싼 비용 (500-1,000만원), 시간 투자 큼
- **영향**: Master Class 경쟁

#### 대학 정규 교육
- **강점**: 학위, 학점 인정
- **약점**: 느린 커리큘럼, 실무 괴리
- **영향**: 학생 시간 경쟁

## 5. 시장 기회

### 5.1 Unmet Needs (충족되지 않은 니즈)

#### 1. 대학 특화 플랫폼 부재
- 현재 플랫폼은 일반 성인 대상
- 학점 연동, 교수 참여 불가능
- 대학 커뮤니티 부재

**기회**: 대학 전용 플랫폼 포지셔닝

#### 2. 협업 학습 경험 부족
- 대부분 플랫폼은 개인 학습
- 팀 프로젝트 지원 미비
- 실시간 협업 도구 없음

**기회**: 협업 중심 플랫폼

#### 3. 창업 지원 생태계 미비
- 학습 후 창업 연결 약함
- 투자 연계 플랫폼 없음
- 법률/회계 지원 부족

**기회**: 창업 All-in-One 플랫폼

#### 4. 단계별 성장 경로 불명확
- 초보자가 중급으로 가는 길 모호
- 레벨 시스템 없음
- 동기부여 부족

**기회**: 게이미피케이션 + 레벨 시스템

### 5.2 시장 진입 전략

#### Blue Ocean Strategy
**제거 (Eliminate)**
- 불필요한 마케팅 비용
- 과도한 강의 콘텐츠
- 복잡한 UI

**감소 (Reduce)**
- 강의 영상 길이 (핵심만)
- 가격 (학생 부담 최소화)
- 기능 복잡도

**증가 (Raise)**
- 실습 비중
- 협업 기능
- 커뮤니티 활동

**창조 (Create)**
- 대학 특화 기능
- 레벨 시스템
- 창업 지원 생태계
- 실시간 코딩 환경

#### First Mover Advantage
- 원광대 내 독점 지위
- 대학 특화 노하우 축적
- 교수-학생 네트워크 선점

### 5.3 시장 진입 장벽

#### 우리의 장벽 (경쟁자 진입 방지)
1. **대학 공식 파트너십**: 신뢰도 및 접근성
2. **커뮤니티 네트워크 효과**: 학생 간 연결
3. **교수진 콘텐츠**: 맞춤형 커리큘럼
4. **데이터 축적**: 학습 패턴 분석

#### 우리가 넘어야 할 장벽
1. **브랜드 인지도**: 신규 플랫폼 신뢰 구축
2. **콘텐츠 부족**: 초기 콘텐츠 확보
3. **사용자 습관**: 기존 플랫폼에서 전환
4. **기술 인프라**: 안정적인 클라우드 환경

## 6. 시장 예측

### 6.1 시장 성장 시나리오

#### 낙관적 시나리오 (Best Case)
- 연간 성장률: 100%
- 3년 후 사용자: 50,000명
- 3년 후 수익: ₩100억

**가정**:
- 바이럴 마케팅 성공
- 주요 대학 파트너십 확보
- 정부 지원금 최대 확보

#### 기준 시나리오 (Base Case)
- 연간 성장률: 50%
- 3년 후 사용자: 30,000명
- 3년 후 수익: ₩70억

**가정**:
- 점진적 성장
- 전라북도 → 전국 확산
- 안정적인 B2B 계약

#### 비관적 시나리오 (Worst Case)
- 연간 성장률: 20%
- 3년 후 사용자: 10,000명
- 3년 후 수익: ₩30억

**가정**:
- 경쟁 심화
- 마케팅 부진
- 기술 문제 발생

- **위험**: 교육 지출 감소
- **대응**: 무료 콘텐츠 확대, 정부 지원금 확보

**2. 보안 사고**
- **위험**: 개인정보 유출, 해킹
- **대응**: 보안 강화, 보험 가입

### 7.3 운영 리스크

**2. 콘텐츠 품질**

### 8.2 전략적 권고사항

#### 단기 (1년)
1. **원광대 내 독점 지위 확보**
   - 학과 공식 채택
   - 교수진 참여 확대
   - 학생 사용률 70% 이상

2. **MVP 검증 및 개선**
   - 사용자 피드백 수집
   - 핵심 기능 최적화
   - PMF (Product-Market Fit) 달성

3. **전라북도 대학 파트너십**
   - 전북대, 군산대 등 접촉
   - 파일럿 프로그램 제안

#### 중기 (2-3년)
1. **전국 확장**
   - 서울/수도권 주요 대학
   - B2B 영업 강화
   - 브랜드 마케팅

2. **수익 모델 고도화**
   - Premium/Pro Tier 최적화
   - 기업 파트너십 확대
   - 정부 지원금 극대화

3. **콘텐츠 확장**
   - Senior/Master 클래스 고도화
   - AI, 블록체인 등 신기술 추가
   - 창업 프로그램 강화

#### 장기 (5년)
1. **해외 진출**
   - 동남아시아 대학
   - 글로벌 플랫폼 파트너십

2. **플랫폼 확장**
   - 기업 교육
   - 일반 성인 교육
   - 청소년 코딩 교육

3. **생태계 구축**
