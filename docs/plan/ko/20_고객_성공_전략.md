# WKU 소프트웨어 Crew Project - 고객 성공 전략

## 1. 고객 성공 (Customer Success) 개요

### 1.1 정의
고객이 플랫폼을 통해 **목표를 달성**하도록 지원하는 모든 활동

### 1.2 목표
- 사용자 만족도 (NPS): > 70
- 리텐션: 4주 리텐션 > 30%
- 유료 전환율: > 10%
- 이탈률: < 5%/월

---

## 2. 고객 여정 (Customer Journey)

### 2.1 Awareness (인지)
**목표**: 플랫폼 존재 알리기

**활동**:
- SNS 마케팅
- 학과 공지
- 입소문

### 2.2 Onboarding (온보딩)
**목표**: 빠르게 첫 성공 경험 (Aha Moment)

**활동**:
- 웰컴 이메일
- 5분 튜토리얼 ("Hello World 배포하기")
- 첫 강의 추천
- 체크리스트 제공

**성공 지표**: 첫 강의 수강 (24시간 내)

### 2.3 Engagement (참여)
**목표**: 정기적 사용 습관 형성

**활동**:
- 주간 학습 리포트 (이메일)
- 학습 스트릭 (연속 출석)
- 알림 (과제 마감, 새 강의)
- 스터디 그룹 매칭

**성공 지표**: 주 3회 이상 로그인

### 2.4 Conversion (전환)
**목표**: 무료 → 유료 전환

**활동**:
- Premium 기능 체험 (1개월 무료)
- 성공 사례 공유
- 한정 할인 (조기 가입자 50% 할인)

**성공 지표**: 유료 전환 8-10%

### 2.5 Retention (유지)
**목표**: 장기 사용자 만들기

**활동**:
- 새로운 콘텐츠 지속 제공
- 멘토링 매칭
- 커뮤니티 활성화
- VIP 프로그램 (Master 클래스)

**성공 지표**: 12주 리텐션 > 20%

### 2.6 Advocacy (옹호)
**목표**: 홍보대사 만들기

**활동**:
- 추천 프로그램 (친구 초대 시 1개월 무료)
- 성공 스토리 인터뷰
- 앰버서더 프로그램

**성공 지표**: K-Factor > 0.5

## 3. 고객 세그먼트별 전략

### 3.1 Junior (입문자)

**니즈**:
- 쉬운 학습
- 동기부여
- 즉각 피드백

**전략**:
- 게이미피케이션 (배지, 레벨업)
- 짧은 강의 (10-15분)
- 스터디 그룹 자동 매칭

### 3.2 Senior (중급)

**니즈**:
- 실전 프로젝트
- 팀 협업
- 포트폴리오

**전략**:
- 프로젝트 기반 학습
- 코드 리뷰
- 포트폴리오 자동 생성

### 3.3 Master (고급 + 창업)

**니즈**:
- 멘토링
- 투자 연계
- 법률/회계 지원

**전략**:
- 1:1 멘토링
- 데모데이 기회
- 전문가 컨설팅

---

## 4. 고객 지원

### 4.1 지원 채널

**1차**: 셀프 서비스
- FAQ (100개 이상)
- Help Center
- 커뮤니티 Q&A

**2차**: 라이브 채팅
- 평일 9-18시
- 평균 응답 시간: 5분

**3차**: 이메일
- support@wkucrew.com
- 24시간 내 응답

**4차**: 전화 (유료 회원)
- Premium/Pro 회원 전용
- 긴급 문의

### 4.2 SLA (Service Level Agreement)

| 우선순위 | 응답 시간 | 해결 시간 |
|----------|-----------|-----------|
| P0 (Critical) | 1시간 | 4시간 |
| P1 (High) | 4시간 | 1일 |
| P2 (Medium) | 1일 | 3일 |
| P3 (Low) | 3일 | 1주 |

---

## 5. 프로액티브 지원 (Proactive Support)

### 5.1 사용자 건강 점수 (Health Score)

**계산 요소**:
- 로그인 빈도 (30%)
- 학습 진행률 (30%)
- 커뮤니티 참여 (20%)
- 과제 제출 (20%)

**점수**:
- 80-100: 건강 (Green)
- 50-79: 주의 (Yellow)
- 0-49: 위험 (Red)

### 5.2 개입 전략

**Yellow (주의)**:
- 자동 이메일: "최근 학습이 멈췄네요. 다시 시작해볼까요?"
- 추천 강의

**Red (위험)**:
- 1:1 이메일 (개인화)
- 전화 연락 (Premium 회원)
- 특별 혜택 (1주일 무료 연장)

## 6. 고객 피드백

### 6.1 수집 방법

**인앱 피드백**:
- 피드백 버튼 (모든 페이지)
- 버그 리포트

**설문조사**:
- NPS (분기별)
- 기능 만족도 (월간)

**사용자 인터뷰**:
- 월 10-20명
- 30분-1시간
- 소정의 사례품 (스타벅스 쿠폰)

### 6.2 피드백 관리

**도구**: Canny, ProductBoard
**프로세스**:
1. 수집
2. 분류 (버그, 기능 요청, 개선)
3. 우선순위화
4. 백로그 추가
5. 개발
6. 피드백 제공자에게 알림

---

## 7. 고객 성공 팀

### 7.1 팀 구성

**Customer Success Manager (CSM)**: 2명
- 역할: 유료 고객 관리, 리텐션, 업셀

**Support Specialist**: 2명
- 역할: 고객 문의 응대

**Community Manager**: 1명
- 역할: 커뮤니티 관리, 이벤트 기획

### 7.2 KPI

**CSM**:
- 담당 고객 리텐션 > 90%
- Upsell (Pro Tier 전환) > 10%
- NPS > 70

**Support**:
- 평균 응답 시간 < 5분
- CSAT (고객 만족도) > 4.5/5.0
- 티켓 해결률 > 95%
