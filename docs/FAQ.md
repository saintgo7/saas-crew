## 계정 및 인증

### Q1. GitHub 계정 없이 가입할 수 있나요?

**A**: 아니요. 현재는 GitHub OAuth 인증만 지원합니다.

**이유**:
- 개발자 플랫폼 특성상 GitHub는 필수 도구
- 프로젝트 GitHub 저장소 연동 편의성
- 안전하고 간편한 인증

**해결 방법**:
1. GitHub 계정을 먼저 생성하세요 (무료)
2. https://github.com 접속
3. "Sign up" 클릭하여 가입
4. 이메일 인증 완료 후 WKU Software Crew 가입

### Q2. 로그인이 안 돼요.

**A**: 다음을 확인해보세요.

**체크리스트**:
- [ ] GitHub 로그인이 정상적으로 되나요?
- [ ] WKU Software Crew에 권한을 승인했나요?
- [ ] 브라우저 쿠키를 차단하고 있지 않나요?
- [ ] 시크릿/비공개 모드를 사용 중이 아닌가요?

**해결 방법**:
1. GitHub 로그아웃 후 다시 로그인
2. WKU Software Crew 권한 재승인
3. 브라우저 캐시 및 쿠키 삭제
4. 다른 브라우저에서 시도

### Q3. 이메일을 변경하고 싶어요.

**A**: GitHub 계정의 이메일을 변경하면 자동으로 반영됩니다.

**방법**:
1. GitHub 설정 (Settings) 이동
2. Emails 섹션에서 이메일 변경
3. WKU Software Crew에 다시 로그인
4. 프로필 페이지에서 변경 확인

### Q4. 계정을 삭제하고 싶어요.

**A**: 현재는 관리자에게 이메일로 요청해야 합니다.

**절차**:
1. admin@wku-crew.com으로 이메일 발송
2. 제목: "계정 삭제 요청"
3. 내용: GitHub 이메일 및 사용자 이름
4. 3-5일 내 처리

**주의사항**:
- 삭제 후 복구 불가능
- 모든 프로젝트, 코스 진도, XP 삭제
- 게시글 및 댓글은 익명으로 전환

**향후 계획**: 설정 페이지에서 직접 탈퇴 기능 추가 예정

### Q5. 세션이 자주 만료돼요.

**A**: 보안을 위해 7일마다 자동 로그아웃됩니다.

**세션 유지 방법**:
- 7일 내 최소 1회 접속하면 자동 연장
- "로그인 상태 유지" 기능 (향후 추가 예정)

**세션 만료 시**:
- 자동으로 로그인 페이지로 이동
- GitHub OAuth 재인증
- 이전 작업 내용은 저장됨

## 레벨 시스템

### Q6. 레벨은 어떻게 올리나요?

**A**: XP(경험치)를 획득하여 레벨을 올립니다.

**주요 XP 획득 방법**:

| 활동 | XP | 비고 |
|------|------|------|
| 챕터 완료 | 10-50 | 난이도에 따라 차등 |
| 코스 완강 | 100-500 | 레벨에 따라 차등 |
| 프로젝트 등록 | 100 | 최초 1회 |
| 프로젝트 업데이트 | 20 | 주 1회 제한 |
| 질문 작성 | 10 | |
| 답변 작성 | 20 | |
| 베스트 답변 | 50 | 추가 보너스 |
| Upvote 받기 | 2 | 받을 때마다 |

**빠르게 레벨업하는 팁**:
1. 코스 완강 (가장 많은 XP)
2. 프로젝트 꾸준히 업데이트
3. 커뮤니티에서 활발히 답변
4. 베스트 답변 받기

### Q7. 레벨업에 필요한 XP는 얼마인가요?

**A**: 레벨이 올라갈수록 필요한 XP가 증가합니다.

**예상 XP 테이블** (참고용, 실제는 조정될 수 있음):

```
Level 1 → 2:     100 XP
Level 2 → 3:     200 XP
Level 3 → 4:     300 XP
Level 4 → 5:     400 XP
Level 5 → 6:     500 XP
...
Level 10 → 11:  1,000 XP (Senior 승급)
...
Level 30 → 31:  5,000 XP (Master 승급)
```

**패턴**: 각 레벨마다 이전 레벨 + 100 XP

### Q8. Junior에서 Senior로 승급하려면?

**A**: Level 11에 도달하면 자동으로 Senior로 승급합니다.

**승급 조건**:
- Level 10 → 11 달성 (XP로만 결정)
- 별도 심사 없음
- 자동 승급

**Senior 승급 시**:
- 랭크 뱃지 변경 (Junior → Senior)
- Senior 레벨 코스 접근 가능
- 멘토링 자격 획득
- 크루 이벤트 우선 참여권

**참고**: Level 31에서 Master로 승급

### Q9. 레벨이 내려갈 수 있나요?

**A**: 아니요. 한번 올린 레벨은 내려가지 않습니다.

**XP 감소 상황**:
- Downvote 받기 (-2 XP)
- 부적절한 게시물 삭제 시
- 과제 재채점으로 점수 하락 시

**하지만**:
- 이미 올린 레벨은 유지
- XP가 음수가 되지 않음
- 최소 0 XP까지만 감소

### Q10. 랭크 뱃지는 어떻게 받나요?

**A**: 레벨에 따라 자동으로 부여됩니다.

**랭크 시스템**:
- **Junior**: Level 1-10
- **Senior**: Level 11-30
- **Master**: Level 31-50

**뱃지 표시 위치**:
- 프로필 페이지
- 게시글/댓글 작성자 이름 옆
- 프로젝트 팀원 목록
- 리더보드

## 프로젝트

### Q11. 프로젝트를 몇 개까지 등록할 수 있나요?

**A**: 제한이 없습니다. 원하는 만큼 등록 가능합니다.

**권장 사항**:
- 포트폴리오용: 3-5개 대표 프로젝트
- 학습용: 제한 없이 등록
- 완성도 높은 프로젝트 위주로 공개

### Q12. 진행 중인 프로젝트도 등록할 수 있나요?

**A**: 가능합니다.

**가시성 설정 활용**:
1. **PRIVATE**: 나만 보기 (개발 시작)
2. **TEAM**: 팀원과만 공유 (협업 중)
3. **PUBLIC**: 완성 후 공개

**장점**:
- 진도 추적
- 동기 부여
- 팀원과 협업

### Q13. 프로젝트 슬러그(URL)를 변경할 수 있나요?

**A**: 아니요. 슬러그는 생성 후 변경할 수 없습니다.

**이유**:
- URL 일관성 유지
- 외부 링크 깨짐 방지

**해결 방법**:
- 잘못 입력한 경우 프로젝트 삭제 후 재생성
- 처음 입력할 때 신중하게 결정

**팁**: 영문 소문자, 하이픈, 숫자만 사용하여 명확하게 작성

### Q14. 프로젝트 쇼케이스에 등록하려면?

**A**: 다음 조건을 충족하고 신청하세요.

**필수 조건**:
- [ ] 가시성이 PUBLIC
- [ ] GitHub 저장소 연동 또는 배포 URL 존재
- [ ] 프로젝트 설명 100자 이상
- [ ] 커버 이미지 등록

**신청 절차**:
1. 프로젝트 편집 페이지
2. "쇼케이스 신청" 버튼 클릭
3. 신청 사유 작성 (선택)
4. 관리자 검토 대기 (3-5일)

**승인 기준**:
- 프로젝트 완성도
- 실제 동작 여부
- 코드 품질
- 독창성 및 유용성

### Q15. GitHub private 저장소도 연동할 수 있나요?

**A**: 현재는 공개 저장소만 가능합니다.

**향후 계획**:
- GitHub App 인증 기능 추가 예정
- private 저장소 연동 지원
- 선택적 정보 공개 (README만 표시 등)

**임시 해결 방법**:
- 배포 URL만 등록
- README를 프로젝트 설명에 복사
- 스크린샷으로 대체

### Q16. 팀 프로젝트는 어떻게 XP가 분배되나요?

**A**: 모든 팀원에게 동일하게 지급됩니다.

**XP 분배 방식**:
- 프로젝트 등록: 각 팀원 100 XP
- 프로젝트 업데이트: 각 팀원 20 XP
- 좋아요 받기: 각 팀원 5 XP

**역할 구분 없음**:
- Owner, Admin, Member 모두 동일
- 기여도와 무관하게 평등 분배

**이유**: 협업 문화 장려

## 코스 및 학습

### Q17. 코스를 순서대로 들어야 하나요?

**A**: 아니요. 자유롭게 선택할 수 있습니다.

**권장 학습 경로**:
- 자신의 레벨에 맞는 코스 선택
- 사전 요구사항 확인 (코스 상세 페이지)
- 기초 → 중급 → 고급 순서 권장

**순차 학습이 필요한 경우**:
- 일부 코스는 챕터 순서 강제
- 이전 챕터 완료해야 다음 챕터 잠금 해제
- 코스 설명에 명시됨

### Q18. 여러 코스를 동시에 들을 수 있나요?

**A**: 가능합니다.

**권장 사항**:
- 2-3개 이내 권장 (집중력 유지)
- 서로 다른 분야 (Frontend + Backend)
- 주당 학습 시간 고려

**예시**:
```
동시 수강 예시:
1. React 기초 (Frontend)
2. Node.js 백엔드 (Backend)
3. Git 마스터 (도구)
```

### Q19. 과제를 꼭 제출해야 하나요?

**A**: 아니요. 선택 사항입니다.

**과제 제출 없이도**:
- 챕터 완료 가능
- 코스 완강 가능
- 기본 XP 획득

**과제 제출 시 혜택**:
- 추가 XP 획득 (50-100 XP)
- 실력 검증
- 피드백을 통한 학습
- 포트폴리오 강화

**권장**: 시간이 허락한다면 제출하세요.

### Q20. 완강한 코스를 다시 들을 수 있나요?

**A**: 가능합니다.

**복습 모드**:
- 완강 후에도 모든 챕터 접근 가능
- 언제든지 다시 학습
- 비디오 재시청 가능

**재학습 시**:
- 추가 XP는 지급되지 않음
- 완강 뱃지는 유지
- 진도율은 100% 유지

**활용 방법**:
- 기억이 안 날 때 복습
- 면접 준비
- 새로운 관점에서 재학습

### Q21. 코스 수료증을 받을 수 있나요?

**A**: 현재는 프로필 뱃지만 제공됩니다.

**현재 제공**:
- 완강 뱃지 (프로필 표시)
- 완강 날짜 기록
- 학습 이력 통계

**향후 계획**:
- PDF 수료증 발급 기능
- 다운로드 및 인쇄
- LinkedIn 공유 기능

## 커뮤니티

### Q22. 질문을 올렸는데 답변이 없어요.

**A**: 다음을 시도해보세요.

**질문 개선**:
1. 제목을 더 구체적으로 수정
2. 코드 예제 추가
3. 환경 정보 제공
4. 시도한 방법 공유

**재게시**:
- 24시간 후 재게시 가능
- 인기 시간대 활용 (오후 6-10시)
- 관련 태그 추가

**다른 방법**:
- 외부 커뮤니티 활용 (Stack Overflow)
- 크루 슬랙/디스코드 채널 (향후 추가)

### Q23. 베스트 답변을 꼭 선택해야 하나요?

**A**: 필수는 아니지만 강력히 권장합니다.

**선택 이유**:
- 답변자에게 보상 (50 XP)
- 다른 크루원에게 유용한 정보
- 커뮤니티 활성화
- 감사 표현

**선택하지 않아도 괜찮은 경우**:
- 문제가 완전히 해결되지 않음
- 모든 답변이 부족함
- 추가 답변을 기다리는 중

**팁**: 충분히 검토한 후 선택하세요 (너무 빨리 선택하지 말기).

### Q24. Downvote를 받았어요. 왜 그런가요?

**A**: 다음 이유일 수 있습니다.

**가능한 원인**:
- 부정확한 정보
- 질문과 관련 없는 답변
- 불친절한 언어
- 중복 답변
- 스팸

**대응 방법**:
1. 댓글로 이유 확인
2. 내용 개선 및 수정
3. 부정확한 정보 수정
4. 친절한 언어로 다시 작성

**예방 방법**:
- 확실한 정보만 답변
- 친절하고 공손한 언어
- 질문을 정확히 이해한 후 답변

### Q25. 오래된 질문에도 답변해도 되나요?

**A**: 가능합니다!

**권장하는 경우**:
- 더 나은 해결책을 알고 있을 때
- 최신 버전의 다른 방법
- 기존 답변의 보완

**주의사항**:
- 기존 답변과 중복되지 않게
- 새로운 정보나 관점 제공
- 버전 정보 명시 (예: "React 18에서는...")

**효과**:
- 오래된 질문도 XP 획득 가능
- 검색으로 찾은 사람에게 도움
- 지식 최신화

## XP 및 보상

### Q26. XP를 잃을 수 있나요?

**A**: 일부 상황에서 XP가 감소할 수 있습니다.

**XP 감소 상황**:
- Downvote 받기: -2 XP
- 부적절한 게시물 삭제: 획득 XP 회수
- 베스트 답변 해제: -50 XP
- 프로젝트 삭제: 관련 XP 회수

**하지만**:
- 레벨은 절대 내려가지 않음
- XP는 0 이하로 떨어지지 않음
- 정상적인 활동으로는 XP 감소 없음

**악의적 행위 시**:
- 스팸, 어뷰징 적발 시
- 모든 관련 XP 회수
- 계정 정지 가능

### Q27. XP를 가장 빨리 얻는 방법은?

**A**: 다음 전략을 추천합니다.

**단기 전략 (1주일)**:
1. Junior 코스 1개 완강 (500 XP)
2. 프로젝트 1개 등록 (100 XP)
3. 매일 답변 1개 (20 XP × 7일 = 140 XP)
4. **총: 740 XP**

**중기 전략 (1개월)**:
1. Senior 코스 2개 완강 (1,000 XP)
2. 프로젝트 3개 등록 (300 XP)
3. 주간 프로젝트 업데이트 (20 XP × 4주 = 80 XP)
4. 매일 답변 2개 (40 XP × 30일 = 1,200 XP)
5. 베스트 답변 3개 (150 XP)
6. **총: 2,730 XP**

**장기 전략 (꾸준함)**:
- 매일 조금씩 활동
- 코스 + 프로젝트 + 커뮤니티 병행
- 양보다 질 (베스트 답변 받기)

### Q28. 보상이나 혜택이 있나요?

**A**: 레벨과 랭크에 따라 다양한 혜택이 있습니다.

**Junior 혜택**:
- 기초 코스 접근
- 커뮤니티 참여
- 프로젝트 등록

**Senior 혜택** (Level 11+):
- Senior 코스 접근
- 멘토링 자격

## 문제 해결

### Q29. 페이지가 로딩되지 않아요.

**A**: 다음을 시도해보세요.

**기본 트러블슈팅**:
1. 페이지 새로고침 (Ctrl+F5 / Cmd+Shift+R)
2. 브라우저 캐시 삭제
3. 다른 브라우저에서 시도
4. 인터넷 연결 확인

**여전히 안 된다면**:
- 서버 점검 중일 수 있음
- 공지사항 확인
- 관리자에게 문의

### Q30. 프로필 사진이 업데이트되지 않아요.

**A**: GitHub 아바타 캐시 문제일 수 있습니다.

**해결 방법**:
1. GitHub에서 프로필 사진 변경
2. 로그아웃 후 다시 로그인
3. 브라우저 캐시 삭제
4. 24시간 후 자동 업데이트

**강제 업데이트**:
- 프로필 편집 페이지에서 "사진 새로고침" 버튼 (향후 추가)

### Q31. XP가 정상적으로 지급되지 않았어요.

**A**: 다음을 확인하세요.

**확인 사항**:
- [ ] 활동이 완료되었나요? (예: 챕터 완료 체크)
- [ ] 중복 활동이 아닌가요? (예: 주간 프로젝트 업데이트 제한)
- [ ] 조건을 충족했나요? (예: 베스트 답변 선정)

**여전히 문제라면**:
1. 활동 이력 확인 (프로필 > 활동 로그)
2. 스크린샷 찍기
3. GitHub Issues에 버그 리포트
4. 관리자에게 이메일

**처리 기간**: 보통 3-5일 내 확인 및 조치

### Q32. 댓글/답변을 삭제했는데 복구할 수 있나요?

**A**: 아니요. 삭제는 되돌릴 수 없습니다.

**삭제 시 영구 삭제**:
- 내용 완전 삭제
- XP도 회수
- 복구 불가능

**주의사항**:
- 삭제 전 신중히 생각하기
- 수정으로 해결 가능한지 확인
- 중요한 내용은 백업

**예방 방법**:
- 삭제 확인 대화상자 꼼꼼히 읽기
- 수정 기능 활용

### Q33. 코스 영상이 재생되지 않아요.

**A**: 다음을 확인하세요.

**문제 해결**:
1. 인터넷 속도 확인
2. 다른 브라우저에서 시도
3. 광고 차단 확장 프로그램 비활성화
4. 브라우저 업데이트

**지원 브라우저**:
- Chrome, Edge, Firefox, Safari 최신 버전
- 모바일: iOS Safari, Android Chrome

**여전히 안 된다면**:
- 동영상 호스팅 문제 (YouTube, Vimeo)
- 관리자에게 문의
- 임시로 텍스트 콘텐츠로 학습

### Q34. 프로젝트 수정이 저장되지 않아요.

**A**: 다음을 시도하세요.

**확인 사항**:
- [ ] "저장" 버튼을 클릭했나요?
- [ ] 필수 항목을 모두 입력했나요?
- [ ] 네트워크 연결이 안정적인가요?
- [ ] 브라우저 콘솔에 에러가 있나요?

**해결 방법**:
1. 내용을 별도 파일에 임시 저장
2. 페이지 새로고침
3. 다시 입력 후 저장
4. 다른 브라우저에서 시도

**버그라면**:
- GitHub Issues에 리포트
- 에러 메시지 스크린샷 첨부
- 브라우저 정보 제공

### Q35. 베스트 답변을 선택했는데 취소할 수 없어요.

**A**: 베스트 답변은 변경 및 취소 가능합니다.

**변경 방법**:
1. 현재 베스트 답변의 체크박스 클릭 (해제)
2. 새로운 답변의 체크박스 클릭 (선택)

**취소 방법**:
1. 베스트 답변 체크박스 클릭 (해제)
2. 베스트 답변 없는 상태로 유지

**주의사항**:
- 이전 답변자의 보너스 XP 회수
- 새로운 답변자에게 XP 지급
- 너무 자주 변경하지 않기
