## 커뮤니티 소개

### 커뮤니티란?

WKU Software Crew의 **커뮤니티**는 Q&A 형식의 지식 공유 플랫폼입니다.

**주요 기능**:
- 기술 질문 및 답변
- 코드 리뷰 요청
- 프로젝트 관련 논의
- 학습 자료 공유

### 커뮤니티의 가치

#### 함께 배우기

**No Stupid Questions (어리석은 질문은 없습니다)**:
- 초보자 질문도 환영
- 모든 질문은 학습의 기회
- 서로 존중하는 문화

#### 지식 공유

**Give & Take**:
- 내가 받은 도움을 다른 이에게
- 답변하면서 더 깊이 학습
- 크루 전체의 실력 향상

#### XP 획득

**활동으로 성장**:
- 질문 작성: 10 XP
- 답변 작성: 20 XP
- 베스트 답변 선정: 50 XP
- Upvote 받기: 2 XP

### 커뮤니티 규칙

#### 1. 존중과 배려

**서로 존중하기**:
- 비난이나 조롱 금지
- 친절하고 공손한 언어 사용
- 초보자도 배려하기

#### 2. 구체적인 질문

**좋은 질문**:
- 명확한 문제 설명
- 시도한 방법 공유
- 관련 코드 첨부

#### 3. 감사 표현

**답변에 감사하기**:
- 답변에 대한 피드백
- Upvote로 감사 표시
- 베스트 답변 선택

## 질문 작성하기

### Step 1: 커뮤니티 페이지 이동

상단 네비게이션에서 **"커뮤니티"** 클릭

```
[스크린샷 자리: 커뮤니티 메인 페이지]
```

### Step 2: 새 질문 작성

우측 상단의 **"질문하기"** 버튼 클릭

### Step 3: 제목 입력

명확하고 구체적인 제목을 작성합니다.

**좋은 제목 예시**:
- "React에서 useState가 비동기로 동작하는 이유가 궁금합니다"
- "Node.js Express에서 CORS 에러 해결 방법"
- "PostgreSQL JOIN 쿼리 성능 최적화 질문"

**나쁜 제목 예시**:
- "도와주세요" (너무 모호함)
- "에러 발생" (구체적이지 않음)
- "이거 어떻게 하나요?" (내용 불명확)

**제목 작성 팁**:
- 핵심 키워드 포함 (React, Node.js, PostgreSQL)
- 문제 상황 간략히 설명
- 50자 이내 권장

### Step 4: 본문 작성

#### 문제 설명

**5W1H 원칙**:
- What (무엇을): 어떤 문제인가?
- Why (왜): 왜 이 질문을 하는가?
- When (언제): 언제 발생하는가?
- Where (어디서): 어느 부분에서 발생하는가?
- How (어떻게): 어떻게 재현되는가?

**좋은 질문 예시**:
```markdown
## 문제 상황

React로 Todo 앱을 만들고 있는데,
useState를 사용해서 할 일을 추가하려고 합니다.

버튼을 클릭하면 상태가 업데이트되는데,
바로 다음 줄에서 console.log를 찍으면
이전 값이 출력됩니다.

## 예상 동작

버튼 클릭 후 즉시 새로운 값이 출력되기를 기대했습니다.

## 실제 동작

이전 값이 출력되고, 다음 렌더링 때 새 값이 나타납니다.
```

#### 코드 첨부

**코드 블록 사용**:

````markdown
```javascript
function TodoApp() {
  const [todos, setTodos] = useState([]);

  const addTodo = () => {
    setTodos([...todos, 'New Todo']);
    console.log(todos); // 여기서 이전 값이 출력됨
  };

  return <button onClick={addTodo}>Add</button>;
}
```
````

**코드 첨부 가이드**:
- 관련 코드만 첨부 (전체 코드는 GitHub 링크)
- 문법 하이라이팅 사용 (javascript, python, java 등)
- 주석으로 문제 부분 표시

#### 환경 정보

**개발 환경 명시**:
```markdown
## 환경

- React: 18.2.0
- Node.js: 20.10.0
- 운영체제: macOS Sonoma
- 브라우저: Chrome 120
```

#### 시도한 방법

**이미 시도한 것들**:
```markdown
## 시도한 방법

1. useEffect를 사용해봤지만 같은 문제 발생
2. setTimeout을 사용했지만 해결되지 않음
3. 구글 검색으로 비슷한 사례를 찾았지만 이해가 안 됨
```

**왜 시도한 방법을 공유하나요?**
- 중복 답변 방지
- 문제 이해도 파악
- 더 정확한 답변 유도

### Step 5: 태그 추가

관련 기술 스택이나 주제를 태그로 추가합니다.

**태그 예시**:
```
[React] [JavaScript] [useState] [State Management]
```

**태그 선택 가이드**:
- 주요 기술 스택 (React, Node.js)
- 구체적 기능 (useState, Express)
- 카테고리 (Frontend, Backend)
- 최대 5개까지 권장

```
[스크린샷 자리: 질문 작성 양식]
```

### Step 6: 미리보기 및 게시

1. **"미리보기"** 탭으로 전환하여 내용 확인
2. 오타나 형식 오류 수정
3. **"질문 게시"** 버튼 클릭

```
✅ 질문이 게시되었습니다!

크루원들의 답변을 기다려주세요.
알림 설정을 켜두면 답변 시 알림을 받습니다.

+10 XP 획득
```

- [ ] 오타나 문법 오류가 없는가?

## 답변 달기

다른 크루원의 질문에 답변하여 지식을 공유하고 XP를 획득할 수 있습니다.

### Step 1: 질문 찾기

#### 최신 질문 보기

커뮤니티 메인 페이지에서 최신 질문을 확인합니다.

```
[스크린샷 자리: 질문 목록]
```

#### 필터링

**태그별 필터**:
- 자신이 아는 기술 스택 선택
- 예: [React], [Node.js], [Python]

**정렬 옵션**:
- 최신순: 방금 올라온 질문
- 답변 없는 질문: 답변이 필요한 질문
- 인기순: 조회수가 많은 질문

#### 검색

특정 키워드로 질문 검색:
```
검색어: "useState 비동기"
```

### Step 2: 질문 읽기

질문을 클릭하여 상세 내용을 확인합니다.

**확인할 사항**:
- 질문자가 무엇을 묻는가?
- 어떤 문제를 겪고 있는가?
- 이미 어떤 시도를 했는가?
- 내가 도움을 줄 수 있는가?

### Step 3: 답변 작성

페이지 하단의 **"답변 작성"** 섹션으로 스크롤합니다.

#### 좋은 답변 작성법

**1. 명확한 해답 제시**

```markdown
## 답변

useState는 비동기로 동작합니다.

setState를 호출한 직후에는 상태가 즉시 업데이트되지 않고,
다음 렌더링 시에 업데이트됩니다.
```

**2. 이유 설명**

```markdown
## 왜 그런가요?

React는 성능 최적화를 위해 여러 상태 업데이트를
배치(batch)로 처리합니다.

setState를 여러 번 호출해도 한 번의 렌더링만 발생하여
효율적입니다.
```

**3. 해결 방법 제시**

````markdown
## 해결 방법

업데이트된 값을 바로 사용하려면 useEffect를 사용하세요:

```javascript
useEffect(() => {
  console.log(todos); // 업데이트된 값 출력
}, [todos]);
```

또는 함수형 업데이트를 사용할 수 있습니다:

```javascript
setTodos(prevTodos => {
  const newTodos = [...prevTodos, 'New Todo'];
  console.log(newTodos); // 새 값 즉시 사용 가능
  return newTodos;
});
```
````

**4. 추가 자료**

```markdown
## 참고 자료

- [React 공식 문서 - useState](https://react.dev/reference/react/useState)
- [Why is setState async?](https://github.com/facebook/react/issues/11527)
```

### Step 4: 답변 게시

**"답변 게시"** 버튼을 클릭합니다.

```
✅ 답변이 게시되었습니다!

+20 XP 획득

질문자에게 알림이 전송되었습니다.
```

### 답변 수정 및 삭제

#### 답변 수정

1. 내가 작성한 답변 하단의 **"수정"** 버튼 클릭
2. 내용 수정
3. **"저장"** 클릭

**수정 이력**:
- 수정 시각 표시
- "edited" 라벨 표시

#### 답변 삭제

1. **"삭제"** 버튼 클릭
2. 확인 대화상자에서 **"확인"** 클릭

**주의**: 삭제 시 받은 XP도 회수됩니다.

## 베스트 답변 선택

질문을 올린 사람은 가장 도움이 된 답변을 **베스트 답변**으로 선택할 수 있습니다.

### 베스트 답변이란?

**베스트 답변**은 질문자가 가장 유용하다고 판단한 답변입니다.

**특징**:
- 질문 상단에 고정 표시
- 초록색 체크마크 표시
- 답변자에게 추가 XP 지급 (50 XP)

### 베스트 답변 선택 방법

#### Step 1: 답변 검토

모든 답변을 읽고 가장 도움이 된 답변을 찾습니다.

**선택 기준**:
- 문제를 정확히 해결했는가?
- 설명이 명확한가?
- 추가 학습 자료를 제공했는가?
- 코드 예제가 유용한가?

#### Step 2: 베스트 답변 선택

선택하려는 답변 우측 상단의 **"베스트 답변"** 체크박스를 클릭합니다.

```
[스크린샷 자리: 베스트 답변 선택 버튼]
```

#### Step 3: 확인

```
✅ 베스트 답변으로 선택되었습니다!

답변자에게 +50 XP가 지급됩니다.
```

### 베스트 답변 변경

한 번 선택한 베스트 답변은 변경할 수 있습니다.

**변경 방법**:
1. 현재 베스트 답변의 체크박스를 다시 클릭하여 해제
2. 새로운 답변의 체크박스를 클릭

**주의**: 이전 답변자의 보너스 XP는 회수되고, 새로운 답변자에게 지급됩니다.

### 베스트 답변 선택 시기

**언제 선택하나요?**
- 답변이 문제를 완전히 해결했을 때
- 여러 답변 중 가장 유용한 것을 찾았을 때
- 충분히 검토한 후 (너무 빨리 선택하지 말기)

**꼭 선택해야 하나요?**
- 선택 사항이지만 강력히 권장
- 답변자에게 보상 및 동기 부여
- 다른 크루원에게도 유용한 정보 제공

## 투표 시스템

질문과 답변에 투표하여 유용성을 표시할 수 있습니다.

### 투표 종류

#### Upvote (추천)

**좋은 질문/답변에 Upvote**:
- 유용한 내용
- 명확한 설명
- 도움이 된 정보

**Upvote 효과**:
- 작성자에게 +2 XP
- 인기 게시물로 상승
- 검색 순위 향상

#### Downvote (비추천)

**부적절한 질문/답변에 Downvote**:
- 불친절한 답변
- 중복 질문
- 스팸 또는 광고

**Downvote 효과**:
- 작성자 XP 감소 (-2 XP)
- 순위 하락
- 일정 수 이상 시 자동 숨김

**주의**: Downvote는 신중하게 사용하세요. 단순히 의견이 다르다는 이유로 Downvote하지 마세요.

### 투표 방법

#### Step 1: 게시물 찾기

투표하려는 질문 또는 답변을 찾습니다.

#### Step 2: 투표 버튼 클릭

게시물 좌측의 화살표 버튼을 클릭합니다.

```
    ▲  (Upvote)
   15  (현재 점수)
    ▼  (Downvote)
```

#### Step 3: 결과 확인

투표가 즉시 반영됩니다.

```
✅ Upvote 완료!

작성자에게 +2 XP가 지급되었습니다.
```

### 투표 취소 및 변경

#### 투표 취소

같은 버튼을 다시 클릭하면 투표가 취소됩니다.

#### 투표 변경

Upvote → Downvote 또는 반대로 변경할 수 있습니다.

**변경 효과**:
- 이전 투표 취소
- 새로운 투표 적용
- XP 차감 및 추가 (총 4 XP 변동)

### 투표 제한

#### 중복 투표 방지

- 같은 게시물에 1인 1표
- 자신의 게시물에는 투표 불가

#### 투표 권한

- 로그인 필수
- Junior 레벨부터 가능

## 댓글 기능

질문과 답변에 짧은 댓글을 달아 추가 의견을 나눌 수 있습니다.

### 댓글과 답변의 차이

#### 답변

**질문에 대한 해답**:
- 문제 해결 방법 제시
- 긴 내용 (보통 여러 문단)
- 베스트 답변 선택 가능
- 20 XP 획득

#### 댓글

**짧은 의견이나 질문**:
- 추가 정보 요청
- 간단한 피드백
- 짧은 내용 (1-2문장)
- 5 XP 획득

### 댓글 작성

#### Step 1: 댓글 입력란 찾기

질문 또는 답변 하단의 **"댓글 추가"** 링크를 클릭합니다.

#### Step 2: 댓글 입력

```
[댓글 입력란]

예시:
"환경 정보를 추가해주시면 더 정확한 답변을 드릴 수 있습니다."
"감사합니다! 덕분에 해결했어요!"
"React 18 버전에서는 이 방법이 더 권장됩니다."
```

#### Step 3: 게시

Enter 키 또는 **"게시"** 버튼을 클릭합니다.

```
✅ 댓글이 추가되었습니다!

+5 XP 획득
```

### 댓글 답글 (대댓글)

다른 사람의 댓글에 답글을 달 수 있습니다.

**답글 작성**:
1. 댓글 우측의 **"답글"** 버튼 클릭
2. 답글 입력
3. 게시

**답글 표시**:
```
김철수: 이 방법이 더 좋을 것 같아요.
  └─ 홍길동: 좋은 의견 감사합니다!
```

### 댓글 수정 및 삭제

**수정**:
1. 내 댓글 우측의 **"수정"** 클릭
2. 내용 수정
3. Enter 또는 **"저장"** 클릭

**삭제**:
1. **"삭제"** 클릭
2. 확인

**주의**: 삭제 시 답글도 모두 삭제됩니다.

## 커뮤니티 활용 팁

### 효과적인 질문 전략

#### 1. 검색 먼저 하기

**중복 질문 방지**:
- 커뮤니티 검색으로 유사 질문 확인
- 구글 검색으로 일반적인 해결책 확인
- 공식 문서 먼저 읽어보기

#### 2. 구체적으로 질문하기

**좋은 질문의 특징**:
- 재현 가능한 예제 코드
- 명확한 문제 설명
- 기대했던 동작과 실제 동작
- 환경 정보

#### 3. 감사 표현하기

**답변자에 대한 예의**:
- "감사합니다" 댓글 달기
- Upvote로 고마움 표시
- 베스트 답변 선택
- 해결 여부 피드백

### 효과적인 답변 전략

#### 1. 친절하게 설명하기

**초보자도 이해할 수 있게**:
- 전문 용어 풀어 설명
- 단계별로 나누어 설명
- 예제 코드 제공

#### 2. 정확한 정보 제공

**검증된 정보만**:
- 확실한 것만 답변
- 추측은 "아마도", "추측컨대" 표시
- 공식 문서 링크 첨부

#### 3. 코드 예제 제공

**동작하는 코드**:
- 복사-붙여넣기만으로 작동
- 주석으로 설명 추가
- 최신 버전 문법 사용

### XP 최대화 전략

#### 높은 XP 활동

**우선순위**:
1. 베스트 답변 받기 (50 XP)
2. 답변 작성 (20 XP)
3. 질문 작성 (10 XP)
4. 댓글 작성 (5 XP)
5. Upvote 받기 (2 XP)

#### 양보다 질

**좋은 답변 하나**가 여러 개의 평범한 답변보다 낫습니다.
- 베스트 답변 = 답변 20개 + 질문 5개 XP
- 질 높은 답변에 집중

#### 꾸준한 활동

**매일 조금씩**:
- 하루 1개 답변 목표
- 주간 2-3개 질문
- 관심 있는 태그 팔로우

## 커뮤니티 에티켓

### 해야 할 것 (Do)

#### 1. 존중하기

- 모든 질문을 존중
- 친절한 언어 사용
- 긍정적인 피드백

#### 2. 구체적으로 작성

- 명확한 제목과 내용
- 관련 코드 첨부
- 환경 정보 제공

#### 3. 감사 표현

- 답변에 감사하기
- 베스트 답변 선택
- Upvote로 고마움 표시

#### 4. 피드백 제공

- 답변이 도움이 됐는지 알리기
- 추가 질문하기
- 해결 여부 공유

### 하지 말아야 할 것 (Don't)

#### 1. 무례한 행동

- 비난이나 조롱
- 공격적인 언어
- 초보자 무시

#### 2. 숙제 대신 해달라는 요청

**허용되지 않음**:
- "이거 대신 해주세요"
- 과제를 그대로 올리고 답 요구
- 스스로 노력하지 않음

**올바른 방법**:
- 시도한 코드와 함께 질문
- 어느 부분이 막혔는지 구체적으로
- 배우려는 의지 보이기

#### 3. 중복 질문

- 이미 답변된 질문 반복
- 검색 없이 질문
- 여러 곳에 동일 질문 복사 붙여넣기

#### 4. 스팸

- 광고성 게시물
- 관계없는 링크
- 무의미한 내용 반복

## 자주 묻는 질문

### Q1. 질문을 올렸는데 답변이 없어요.

**A**: 다음을 시도해보세요.
1. 제목과 내용을 더 구체적으로 수정
2. 코드 예제 추가
3. 관련 태그 추가
4. 인기 시간대에 재게시 (오후 6-10시)

### Q2. 베스트 답변을 꼭 선택해야 하나요?

**A**: 필수는 아니지만 강력히 권장합니다. 답변자에게 보상을 주고, 다른 크루원에게도 유용한 정보를 제공합니다.

### Q3. 답변을 잘못 작성했어요. 어떻게 하나요?

**A**: 수정 또는 삭제할 수 있습니다. 답변 하단의 "수정" 버튼을 클릭하여 내용을 고치세요.

### Q4. Downvote를 받았어요. 왜 그런가요?

**A**: 다음 이유일 수 있습니다.
- 부정확한 정보
- 질문과 관련 없는 답변
- 불친절한 언어
- 댓글로 설명을 요청하거나 내용을 개선하세요.

### Q5. 같은 질문에 여러 개의 답변이 있어요. 어떤 걸 선택하나요?

**A**: 다음 기준으로 선택하세요.
- 가장 명확하게 문제를 해결한 답변
- 이해하기 쉬운 설명
- 유용한 추가 자료 제공
- 여러 답변이 도움이 됐다면 댓글로 각각 감사 표현

### Q6. 오래된 질문에도 답변해도 되나요?

**A**: 가능합니다! 새로운 정보나 더 나은 해결책이 있다면 언제든지 답변해주세요.
